# cc-bar 需求草案（粗略版）

> 第一阶段需求文档，只列大方向和边界，细节留到后面单独拆。
> 目标：一个**原生 macOS 菜单栏小工具**，统一查看 Codex 和 Claude Code 的额度与用量。

---

## 0. 产品定位

- **形态**：常驻菜单栏的小工具（Accessory App，不占 Dock）。
- **平台**：macOS（最低版本待定，初步定 macOS 14 Sonoma+，以便用最新 SwiftUI / MenuBarExtra）。
- **设计**：完全遵循 macOS 最新 HIG（Human Interface Guidelines），尽量使用系统默认控件、系统字体、系统配色，不做自定义视觉风格。
- **一句话价值**：装上它 → 不打开 CLI 也能随时看到 Codex/Claude 还剩多少 5 小时额度、本周/本月花了多少钱。

---

## 1. 账号自动识别

- 自动扫描本机的 Codex / Claude Code 凭据：
  - Codex：`~/.codex/auth.json`（OAuth 模式）
  - Claude：`~/.claude/.credentials.json` 或 macOS Keychain `Claude Code-credentials`
- 识别后展示账号基本信息（邮箱 / plan / 账户 ID 等能从 token 里解出来的部分）。
- **用户可在设置里勾选「显示哪些应用」**（Codex / Claude / 两者）。未勾选的应用全程不展示、不查询。
- 不做 App 内登录流程（一期）。前提：用户已经在终端跑过 `codex login` / `claude` 登录。

---

## 2. 菜单栏图标显示

菜单栏的图标右侧可以**显示一段文字**（百分比），具体显示什么由用户在设置里勾选：

- 显示哪个应用：Codex / Claude / 两者同时（同时显示时按顺序拼接，例如 `C:34% · K:60%`，缩写待定）。
- 显示哪种额度：
  - 5 小时窗口剩余百分比（默认）
  - 周窗口剩余百分比
  - 或者两者都显示
- 都不勾就只显示纯图标（仅图标）。
- 图标用 SF Symbol，自动适配深色 / 浅色菜单栏（模板图标）。

---

## 3. 点击菜单栏弹出小面板

- 点击菜单栏图标 → 弹出一个小面板（Popover 风格，系统默认外观）。
- **展示顺序固定：先 Codex，再 Claude**。
- 每个应用一个区块，展示（精简版，详见 [04-界面布局.md](04-界面布局.md) §2）：
  - 5 小时窗口：已用百分比 + 重置时间倒计时
  - 周窗口：已用百分比 + 重置时间倒计时
  - 今日 cost（本地 JSONL 解析，Codex / Claude 分开）
- 底部按钮一行：**刷新 · 设置 · 退出**（无「详细」入口，详细统计走"设置→主窗口"统计 tab）。
- 账号信息（邮箱 / plan）、本周 / 本月明细、月度额外消费等都挪到主窗口里展示。

---

## 4. 桌面悬浮窗（可选开关）

- 设置里可勾选「在屏幕上显示悬浮窗」。
- 默认两行：
  - 第一行：Codex 5 小时剩余百分比
  - 第二行：Claude 5 小时剩余百分比
- 用户可在设置里关掉其中一个 → 只显示一行。
- 样式：无标题栏、半透明、圆角、不抢焦点、可拖动、置顶可选（默认开）。
- 位置记忆：拖到哪里下次还在哪里。

---

## 5. 自动后台刷新

- 后台周期性查询两边的额度 API：
  - Codex：`GET https://chatgpt.com/backend-api/wham/usage`
  - Claude：`GET https://api.anthropic.com/api/oauth/usage`
- **本地 JSONL 同样周期性增量扫描**（增量算 token / 花费）。
- 用户可在设置里配置：
  - 额度 API 刷新间隔（默认 5min，可选 1min / 5min / 10min / 30min / 关闭）
  - JSONL 扫描间隔（默认 30s）
- 手动刷新按钮始终可用。
- 为降低 Codex / Claude Code 非公开额度接口的限流风险：
  - 同一应用 60s 内重复触发刷新时，默认复用上次成功数据，不重复打 API。
  - 遇到 `429` 后，该应用至少 10min 内不再请求，继续显示上次成功数据。
  - 手动刷新可绕过普通 60s 节流，但仍遵守 `429` 退避。
- 成功获取的额度快照要落盘到 `~/Library/Application Support/com.nanvon.ccbar/quota-cache.json`。
- App 启动时先读取本地额度缓存，让用户立刻看到上次数据；不在启动瞬间打额度接口，等待周期刷新或用户手动刷新。
- 失败时不弹错误窗、不清空已有数据，仅在面板内用小字提示当前请求失败 / 数据已过期。

---

## 6. 历史用量统计（主窗口）

- 从 Popover 底部「设置」按钮进入主窗口，主窗口顶部 TabView 切到「统计」tab（默认就在这个 tab）。详见 [04-界面布局.md](04-界面布局.md) §3。
- 支持按时间范围筛选 token 用量和花费，**预设按钮**：
  - 今天
  - 本周
  - 本月
  - 最近 7 天
  - 最近 30 天
  - 全部
- 自定义范围：两个 DatePicker 选起止日期。
- 展示内容（粗略，细节后定）：
  - 汇总卡片：总 input / output / cache_read / cache_create token、总花费 USD
  - 按应用拆分（Codex vs Claude）
  - 按模型拆分（gpt-5 / claude-opus-4 / claude-sonnet-4 …）
  - 按日趋势图（可选，一期可以先用列表）
- 数据来源：纯本地 JSONL 解析的结果（额度 API 不提供历史数据）。

---

## 7. 设置项汇总（一期）

| 分类 | 项 |
|---|---|
| 账号 | 勾选要显示的应用（Codex / Claude） |
| 菜单栏 | 显示哪个应用、显示哪种额度（5h / 周 / 两者） |
| 悬浮窗 | 是否开启、显示哪些行 |
| 刷新 | API 刷新间隔、JSONL 扫描间隔 |
| 通用 | 开机自启、关于 |

---

## 8. 一期不做的（明确划界）

- 不做 App 内的 OAuth 登录流程。
- 不做浏览器 Cookie 自动导入（CodexBar 那套 SweetCookieKit 兜底）。
- 不做 WebView 抓 dashboard DOM。
- 不做 CLI PTY 兜底解析。
- 不做 Gemini / Copilot / 国产 Token Plan。
- 不上 Mac App Store（开沙盒后读不到 `~/.claude`、`~/.codex`，划不来）。
- 不做组织级 Admin API。
- 不做通知（额度告警一期不上，二期再说）。

---

## 9. 风险 / 待确认

1. **接口不稳定**：`wham/usage`、`/api/oauth/usage` 都是非公开 API，字段可能随时改 → 需要降级容错和清晰的错误提示。
2. **Keychain 弹窗**：首次读 `Claude Code-credentials` 系统会弹「允许访问钥匙串」一次。需要文案引导。
3. **凭据过期**：Codex 的 `access_token` JWT 会过期，需要用 `refresh_token` 自己续期。Claude 这边一期先假设 token 有效，过期就提示用户重新登录。
4. **JSONL 量大**：用户长时间使用后日志几百 MB 起步 → 必须做增量扫描 + 缓存。
5. **价格表**：内置一份硬编码（参考 cc-switch 的 seed），后续再考虑从 models.dev 同步。
6. **签名公证**：发布前需要 Apple Developer 证书（99 USD/年）。开发期可以不签名，README 写「右键打开」。
