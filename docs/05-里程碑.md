## M1：凭据读取

**目标**：App 启动时能从本机识别出 Codex 和 Claude 的账号，结果暂时打到 `print()` / 通过 AppState 暴露给 UI 兜底显示。

**涉及**（在现有 Xcode 工程里**新增**这些文件）

- `Core/Credentials/CodexAuth.swift`：读 `~/.codex/auth.json`，解 JWT 拿邮箱 / plan / `account_id`
- `Core/Credentials/ClaudeAuth.swift`：读 `~/.claude/.credentials.json`；不存在则调 `security find-generic-password -s "Claude Code-credentials" -w` 读 Keychain
- `Core/Storage/Settings.swift`：UserDefaults 封装（先建好，内容空）
- 改 `Core/AppState.swift`：加 `codexAccount` / `claudeAccount` 两个 `@Observable` 属性
- 改 `CCBarApp.swift`：启动时 `Task { ... }` 调一次两个 Auth，结果灌进 AppState

**验收**

- ⌘R 跑起来，Xcode 控制台能看到：
  - Codex：邮箱 / plan / token 是否过期
  - Claude：邮箱 / plan / 凭据来源（文件 or Keychain）
- 首次读 Keychain 弹一次系统授权窗，允许后能正常读取
- Popover 占位区可以临时显示账号信息（验证 AppState 通了）

**风险**

- Keychain 弹窗文案；Codex JWT 字段名可能与预期不同

## M2：额度 API 客户端

**目标**：拉到两边的 5h / 1w 接口原始已用百分比 + 窗口重置时间，UI 统一展示为剩余百分比。

**涉及**

- `Core/Quota/CodexQuotaClient.swift`：`GET https://chatgpt.com/backend-api/wham/usage`，带 Codex access_token
- `Core/Quota/ClaudeQuotaClient.swift`：`GET https://api.anthropic.com/api/oauth/usage`，带 Claude token
- Codex token 刷新逻辑（refresh_token → 新 access_token），原子写回 `~/.codex/auth.json`
- 统一返回结构 `QuotaSnapshot { fiveHourUsedPct, fiveHourResetAt, weekUsedPct, weekResetAt }`
- 灌进 `AppState.codexQuota` / `claudeQuota`

**验收**

- 启动 App 后控制台能打印两边的 5h/1w 百分比 + 倒计时
- Codex token 过期场景下能自动刷新（手动把 expires_at 改到过去再跑一次）
- API 失败时返回 `Result.failure(...)`，不抛崩

**依赖**：M1

## M3：菜单栏文字 + 调度器

**目标**：打开 App 就在菜单栏看到图标 + 百分比文字，自动周期刷新。

**涉及**

- `CCBarApp.swift`：已有 `MenuBarExtra`，不动结构
- `MenuBar/MenuBarLabel.swift`：从只显示 SF Symbol，扩成「图标 + 右侧百分比文字」
- 新增 `MenuBar/MenuBarLabelFormatter.swift`：根据 AppState + 设置拼字符串（如 `C:34% · K:60%`）
- 新增 / 改造 `Core/Scheduler.swift`：周期任务，M3 默认额度 API **2 分钟**一次（正式默认可改回 5 分钟）
- `Scheduler` 对 Codex / Claude **分别**维护刷新状态：
  - `lastSuccessAt`：60s 内重复触发刷新时直接复用缓存
  - `nextEarliestRefreshAt`：收到 `429` 后设置为 `now + 10min`
  - `lastError`：UI 展示错误，但不清空已有快照
- 新增 `Core/Storage/QuotaCache.swift`（或同等文件）：把成功的 `QuotaSnapshot` 持久化到 `~/Library/Application Support/com.nanvon.ccbar/quota-cache.json`
- App 启动顺序改成：先同步读 `quota-cache.json` 灌入 AppState → 再读凭据 → 等周期 tick 或用户手动刷新再请求网络
- 改 `AppState.refreshQuotas()` / 单应用刷新逻辑：请求失败时保留内存里的旧 `QuotaSnapshot`，只更新 `codexQuotaError` / `claudeQuotaError`
- `Core/Quota/ClaudeQuotaClient.swift` 请求头补 `User-Agent: claude-code/<ver>`，对齐 CodexBar 的低风险做法
- Claude fallback：仅在「手动刷新 + API 失败 + 没有可展示 Claude 快照」时尝试本机 `claude /usage`，成功后以 `cliFallback` 来源写入同一缓存；不做浏览器 Cookie fallback
- 显示规则：先用硬编码（"Codex + Claude · 5h"）跑通；M7 再接到设置

**验收**

- App 启动后菜单栏出现 SF Symbol 图标
- 图标右侧显示形如 `C:34% · K:60%`
- 数字会随后台刷新自动更新（M3 默认 2 分钟；正式发布默认可改回 5 分钟）
- 重启 App 后，即使网络还没返回，也能先显示上次成功写入的额度缓存
- 额度请求失败时，菜单栏 / Popover 继续显示旧数据，并用错误文案标记当前刷新失败
- 60s 内连续打开 Popover / 连点刷新，不会对同一应用重复发 API 请求
- 模拟或遇到 `429` 后，10 分钟退避期内不再请求该应用，UI 继续展示缓存
- Claude 无缓存且 API 失败时，点击刷新最多触发一次 CLI fallback；fallback 失败也不会清空菜单栏槽位
- 深色 / 浅色菜单栏图标都正常

**依赖**：M2

## M4：总览 Popover

**目标**：点击菜单栏图标，弹出 [04-界面布局.md](04-界面布局.md) §2 描述的面板。

**涉及**

- 改造现有 `MenuBar/PopoverRootView.swift`（已是占位骨架，把"待接入业务模块"那段换成真实视图）：
  - Codex 区块 + Claude 区块

- 双区块、百分比、倒计时数字正确
- 三个按钮都有点击反馈（设置点了没反应也行）

## M5：JSONL 扫描 + Pricing + 聚合

**目标**：把本地日志算成可用的"今日 cost / 按日 / 按模型"数字，Popover 里今日 cost 不再是 `-`。

**涉及**

- `Core/Usage/ClaudeJSONLScanner.swift`：扫 `~/.claude/projects/**/*.jsonl`
- `Core/Usage/CodexJSONLScanner.swift`：扫 `~/.codex/sessions/**/*.jsonl`（路径以实际为准）
- `Core/Usage/Pricing.swift`：硬编码模型价格表（参考 cc-switch 的 seed）
- `Core/Usage/UsageAggregator.swift`：按日 / 按模型 / 按应用聚合
- `Core/Storage/ScanCache.swift`：记录每个文件的扫描偏移 + 累计 token，避免每次全量扫
- 扫描默认间隔 30s，跑在后台队列

**验收**

- Popover 的"今日 cost"显示真实数字（Codex / Claude 分开）
- 跑两次 App，第二次启动时 ScanCache 复用，扫描时间显著缩短
- 故意给某个 JSONL 末尾追加几行，下次扫描能增量识别

**依赖**：M4

## M6：主窗口 - 替换 Settings scene + 统计 tab

**目标**：从 Popover 的"设置"按钮能打开主窗口，主窗口顶部 TabView 两个 tab（统计 / 设置），默认在统计 tab。

> **结构变更**：当前 `CCBarApp.swift` 用的是 SwiftUI 标准的 `Settings { SettingsRootView() }` scene（按 ⌘, 打开的那种）。这一步要**改成普通 `Window` scene**，里面套 `TabView`。原 `Settings/SettingsRootView.swift` 当作设置 tab 的内容搬进去，等 M7 再细化。

**涉及**

- 改 `CCBarApp.swift`：把 `Settings { ... }` 换成 `Window("CCBar", id: "main") { MainWindowRootView() }`
- 新增 `Main/MainWindowRootView.swift`：`TabView`，两个 tab：统计 / 设置；默认选中"统计"
- 新增 `Main/StatsView.swift`：
  - 顶部时间范围选择器（今天 / 本周 / 本月 / 7 天 / 30 天 / 全部 / 自定义）
  - Codex 汇总卡片 + Claude 汇总卡片（input / output / cache_read / cache_create / 总花费）
  - 按模型拆分列表（每个应用一组）
  - 趋势先用列表形态，二期再上 Chart
- 设置 tab 暂时**直接嵌入** `SettingsRootView()`（M7 重写它）
- 改 Popover：「设置」按钮从 `openSettings()` 改成 `openWindow(id: "main")`

**验收**

- 点 Popover 设置 → 主窗口出现，默认在统计 tab
- 顶部 tab 可切到设置（暂时显示原来 4 个旧 tab，不奇怪，M7 修）
- 切换时间范围数字正确变化
- 汇总卡片 Codex / Claude 分开，不合计

**依赖**：M5

## M9：打磨 + 发布准备

**目标**：能对外发的 0.1。

**清单**

- **状态色**：<20% 橙、=0 红（[03-设计风格.md](03-设计风格.md) §0）
- **等宽数字**：所有百分比 / 倒计时 / cost 加 `.monospacedDigit()`

- **识别色**：Codex 石墨灰、Claude 桃橙
- **错误降级**：API 失败在 Popover 底部小字提示，不弹窗、不崩
- **首次 Keychain 授权**：加引导文案
- **开机自启**：跑通 `SMAppService.mainApp.register()`
- **App 图标**：先用 SF Symbol 占位（二期换正式 .icns）
- **README + 截图**
- **签名 / 公证**：有证书就签，没有就 README 写"右键打开"

**验收**

- 自己装一次，从启动到看到数据全流程顺畅
- 关掉网络 → 不崩、有提示
- 关掉 Codex / Claude 任一登录 → 不崩，区块消失

**依赖**：M8
