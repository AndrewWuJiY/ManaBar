# 打包发布

本文只记录 cc-bar 当前工程的打包步骤。工程产物名为 `CCBar.app`，版本号来自 Xcode build settings：`MARKETING_VERSION = 0.1.0`，`CURRENT_PROJECT_VERSION = 1`。

## 1. 打包前检查

1. 用完整 Xcode 打开 `ccbar.xcodeproj`，不要只安装 Command Line Tools。
2. 在 Xcode 左侧选中项目 `ccbar`，进入 target `ccbar` 的 `Signing & Capabilities`。
3. `Team` 选择你的 Apple Developer 账号。
4. `Bundle Identifier` 保持 `com.nanvon.ccbar`。如果这个 id 已被占用，改成你自己的唯一 id，例如 `com.yourname.ccbar`。
5. 顶部 scheme 选择 `ccbar`，运行目标选择 `Any Mac` 或本机 Mac。

## 2. 只给自己用

1. Xcode 菜单选择 `Product` -> `Archive`。
2. Archive 完成后会打开 Organizer。
3. 选择刚生成的 archive，点击 `Distribute App`。
4. 选择 `Copy App`，导出 `CCBar.app`。
5. 把 `CCBar.app` 拖到 `/Applications` 后启动。

这种方式适合自己电脑使用。发给别人时，对方可能会遇到 Gatekeeper 的“无法验证开发者”提示。

## 3. 发给别人用

需要 Apple Developer 账号和 Developer ID 证书。

1. Xcode 菜单选择 `Product` -> `Archive`。
2. Organizer 中点击 `Distribute App`。
3. 选择 `Developer ID`。
4. 选择 `Upload` 让 Apple notarize。
5. notarization 成功后导出 `CCBar.app`。
6. 压缩成 zip 后发布，例如：

```bash
ditto -c -k --keepParent CCBar.app CCBar.app.zip
```

## 4. 命令行归档

如果要用命令行，先确认 `xcodebuild` 指向完整 Xcode：

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

归档：

```bash
xcodebuild archive \
  -project ccbar.xcodeproj \
  -scheme ccbar \
  -configuration Release \
  -archivePath build/CCBar.xcarchive
```

Developer ID 导出：

```bash
xcodebuild -exportArchive \
  -archivePath build/CCBar.xcarchive \
  -exportPath build/export \
  -exportOptionsPlist ExportOptions-DeveloperID.plist
```

导出成功后，app 在 `build/export/CCBar.app`。

## 5. 常见问题

- 如果 `xcodebuild` 提示 active developer directory 是 Command Line Tools，说明没有切到完整 Xcode，按第 4 节先执行 `xcode-select`。
- 如果签名失败，先回到 Xcode 的 `Signing & Capabilities` 检查 Team 和 Bundle Identifier。
- 如果只是自己使用，不需要 notarization；如果发给别人长期使用，建议走 Developer ID + notarization。
