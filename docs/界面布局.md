## 1. Popover

### 1.1 整体规格

| 项 | 值 |
|---|---|
| 宽 | 340pt(固定) |
| 高 | auto;双服务 ~320pt,单服务 ~180pt,零服务 ~110pt(提示空状态) |
| 圆角 | 18pt(`NSPopover` 系统提供,不自绘) |
| 材质 | `.popover`(系统) |
| 锚点 | 菜单栏图标下方,7pt arrow notch 朝上 |
| 出现动画 | 系统默认(`MenuBarExtra(.window)` 提供) |

### 1.2 结构总览(top → bottom)

```
┌───────────────────────────────────────────────────────────┐
│ Header                                       14/12/12pt  │
│   "Usage"  13pt 600                                       │
│   "用量 · refreshed 32s ago"  11pt secondary              │
│                          ● [📊] [↻] [⚙] [⏻]               │
├───────────────────────────────────────────────────────────┤  ← 0.5pt separator full
│ Codex block                                  14/16pt     │
│   ▢ Codex                              resets in 2h 18m   │
│     OpenAI · Plus                                         │
│                                                           │
│   ╭─────╮  Weekly · 周额度                          31%   │
│   │ 42% │  ▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░  (5pt bar)        │
│   │  5H │                                                 │
│   ╰─────╯  184k / 440k        $12.40                      │
│            tokens used        this week                   │
├───────────────────────────────────────────────────────────┤  ← 0.5pt separator inset 16
│ Claude Code block(同上,桃橙色)                           │
└───────────────────────────────────────────────────────────┘
```

> 与原型差异:已移除独立 footer。原 footer 上的「Open Statistics」按钮 和「refresh」按钮合并为 header 右侧的一级图标按钮,刷新进度通过刷新图标自身旋转表达,不再单独画进度条。

### 1.3 Header

- **容器**:HStack(spacing 4),padding `14pt 上 / 12pt 左右 / 12pt 下`(到 separator)。
- **左侧**(VStack,line-height 1.1):
  - `Usage` — 13pt .semibold
  - `用量 · refreshed 32s ago` — 11pt secondary
    - `refreshed {n}s ago` / `{n}m ago` / `{n}h ago`,取 Codex 与 Claude 中**最近**一次成功时间;无成功时显示 `用量 · 等待数据`。
- **右侧**(HStack,spacing 4):
  1. **status dot** — 7pt 圆点(live/stale/offline),颜色按 设计风格 §4.3,hover tooltip `Live · 在线` 等。
  2. **Open Statistics** — `chart.bar.xaxis` 13pt,按钮 `26pt × 22pt` 圆角 5pt,hover 浅灰背景(`Color.primary.opacity(0.08)`)+ 手型光标。点击 `openWindow(id: "main")`,Popover 关闭。
  3. **Refresh** — `arrow.clockwise` 13pt,同尺寸按钮。点击触发 `appState.refreshNow()`,图标旋转 360° / 700ms ease-in-out,刷新中按钮 disabled。
  4. **Settings** — `gearshape` 13pt,同尺寸按钮。点击打开主窗口(当前与 Statistics 共用入口,落到 `appState.mainTab` 当前所记的 tab;⌘, 命令会显式切到设置 tab)。
  5. **Quit** — `power` 13pt,同尺寸。点击 `NSApp.terminate(nil)`,hover tooltip `Quit · 退出`。

> 设计依据:用户反馈一级操作(设置 / 刷新 / 统计 / 退出)藏在 kebab + footer 里不直观,统一拉到顶部图标行,降低点击层级。Quit 早期收在 `ellipsis` 菜单需要点两次,后续也拉平为一级图标。

- **separator**:0.5pt 全宽,`.separatorColor`。

### 1.4 服务 block(Codex / Claude 共用结构)

整个 block 用 `14pt 上下 / 16pt 左右` padding。

#### 1.4.1 Service header row(HStack,gap 9pt)

| 元素 | 规格 |
|---|---|
| 服务 tile | 22pt × 22pt squircle(6pt 圆角),服务识别色填充,白色 logo 14pt 居中。详见 设计风格 §11.2。 |
| 服务名 + plan(VStack,line-height 1.08) | `Codex` 13pt .semibold kerning -0.1<br>`OpenAI · Plus` 10.5pt secondary(plan 取自 `appState.codexAccount?.planType`,缺失时只显示 `OpenAI`) |
| Spacer | — |
| Reset hint | `resets in 2h 18m` 11pt tabular secondary。取自 5h window 的 `resetsAt`。<br>无数据时显示 `reset unknown`。 |

#### 1.4.2 Body row(HStack,gap 14pt,marginTop 12pt)

**左栏:进度环**(56pt 直径,5.5pt 描边,详见 设计风格 §11.3)

- 进度 = `fiveHour.remainingPercent / 100`,起点 12 点钟,顺时针(剩余越多环越满)。
- 中央 VStack(spacing 1,line-height 1):
  - 百分比 14pt .semibold tabular(状态色)
  - `5H` 8pt tertiary letterSpacing 0.2
- 状态色规则按 设计风格 §4.3。

**右栏:weekly + stats**(VStack,flex 1)

1. **Weekly 标题行**(HStack space-between,marginBottom 4pt):
   - 左:`Weekly · 周额度` 10.5pt secondary
   - 右:`31%` 10.5pt tabular(默认色,**不**着状态色)
2. **Weekly 条**:高 5pt,圆角 2.5pt,服务色填充 + `.quaternarySystemFill` 轨道。进度 = `weekly.remainingPercent / 100`。
3. **Claude Design 条**:仅 Claude block 渲染，位于 Weekly 条下方；左侧标签显示 `Design`，8.5pt .semibold，右侧百分比/重置时间/进度条样式与 Weekly 行一致。旧缓存或 API 未返回 `weeklyDesign` 时显示 `--%` 与空轨道。
4. **Stats 双列**(HStack,gap 14pt,marginTop 9pt):
   - 左:`184k / 440k` 11pt tabular .medium primary(主)+ `tokens used` 9.5pt tertiary(副,在主下方,无 ·)
   - 右:`$12.40` 11pt tabular .medium primary + `this week` 9.5pt tertiary

> Prototype 这里**没有**写中文副标(`tokens used` / `this week` 是英文单语)。但我们用 设计风格 §5.4 词表:实际渲染 `tokens used · 已用 tokens` 二行 / `this week · 本周` 二行,字号顶部 11pt,中文 9.5pt tertiary 紧贴下一行。

#### 1.4.3 Block 间分隔

- Codex 与 Claude 之间:0.5pt 横线,**左右各 inset 16pt**(prototype 是 `margin: 0 16px`)。
- Claude block 之后:无 footer,Popover 在此结束(由系统的 `NSPopover` 圆角承接底边)。

#### 1.4.4 空状态 / 错误状态

- `appState.codexQuota == nil` 且 `codexQuotaError == nil`:整个右栏替换为 `Waiting for data · 等待数据` caption secondary 居左;环显示空轨道 + 中央 `--%`。
- `codexQuotaError != nil`:右栏底部追加一行 11pt secondary,显示错误首行(`shortError` 截断 110 字)。环按最后一次成功快照继续显示(`appState.codexQuota` 不清空)。
- 服务在 Settings 里被关闭(`showCodex == false`):**整个 block 不渲染**。
- 两个服务都关闭:Popover 只显示 header + 中部一段 `No services enabled · 未启用任何服务` caption,提示去 Settings。

### 1.5 Footer(已废弃)

> 原方案的 footer(左侧 `Open Statistics` 按钮 + 右侧 refresh + kebab)已**完全移除**。所有一级操作上移到 §1.3 Header 的图标行,刷新进度通过 header 刷新图标自身旋转 + disabled 表达。本节保留仅为历史参考,新实现不要再引入 footer。

### 1.6 交互

| 触发 | 行为 |
|---|---|
| 点 header `chart.bar.xaxis` | `openWindow(id: "main")`,Popover 关闭 |
| 点 header `arrow.clockwise` | 触发 `appState.refreshNow()`,旋转 360° / 700ms,最低保持 0.5s 防闪 |
| 点 header `gearshape` | 打开主窗口(后续可定位到 Settings tab) |
| 点 header `power` | `NSApp.terminate(nil)`,退出 App |
| 点 service block 任意空白 | 暂不响应(后续可定位主窗口到对应服务 scope) |
| 点 Popover 外 | Popover 关闭(系统默认) |
| Esc | Popover 关闭(系统默认) |
| ⌘R(focus 在 Popover) | 触发 refresh |
| 鼠标悬停任意图标按钮 | 浅灰圆角背景 + 手型光标(详见 设计风格 §11.9) |

### 1.7 数据来源对照

| UI 字段 | 数据源 |
|---|---|
| 5h 百分比(环) | `appState.{codex,claude}Quota?.fiveHour.remainingPercent` |
| 5h reset hint | 同上 `.resetsAt` |
| Weekly 百分比 + 条 | `appState.{codex,claude}Quota?.weekly.remainingPercent` |
| Claude Design 百分比 + 条 | `appState.claudeQuota?.weeklyDesign.remainingPercent` |
| `{used}k / {cap}k tokens` | `weekly.used / weekly.cap`(格式化见 `StatsFormatter.token`) |
| `${spend} this week` | 周 spend(从 `UsageAggregator` 取本周 cost 总和) |
| Header refreshed time | 两服务 `refreshState.lastSuccessAt` 取 max |
| 状态点 live/stale/offline | 现有 `headerState` 三态判定 |
| Plan 副标(`OpenAI · Plus`) | `appState.{codex,claude}Account?.planType / subscriptionType` |
| 服务 tile logo | `Resources/Assets.xcassets/codex.svg / claude.svg` |

## 2. Floating HUD

### 2.1 整体规格(默认变体:Two-row pill)

| 项 | 值 |
|---|---|
| 宽 | auto,最小 168pt |
| 高 | auto(两行 pill 约 50–60pt) |
| 圆角 | 14pt |
| 材质 | `.hudWindow`(`NSVisualEffectView`,`blendingMode: .behindWindow`) |
| 背景 alpha | 浅 0.58 / 深 0.62(material 自带) |
| 阴影 | `0 10 28 rgba(0,0,0,0.28)` + inset 0.5pt `rgba(255,255,255,0.1)` 高光(material 自带 + SwiftUI 叠一层 fallback 见 设计风格 §7) |
| 内边距 | 10pt 上下 / 14pt 左右 |

### 2.2 内容结构(variant 1)

VStack,spacing 7pt。每行结构如下:

```
[●]  ████████░░░░░░░░░░  [42%]
[●]  ███████████████░░░  [78%]
 18pt    flex,4pt bar     34pt
```

| 元素 | 规格 |
|---|---|
| 左侧 logo tile | `ServiceTile` 18pt 圆角 5pt,服务色填充 + 内嵌 logo 12pt(`codex.svg` / `claude.svg`),logo 加载失败时回退字母 `C` / `K` |
| 中间 bar | 高 4pt,圆角 2pt,服务色填充 + `.quaternarySystemFill` 轨道,flex 1(最小宽 56pt) |
| 右侧百分比 | 13pt .semibold tabular kerning -0.3,**服务色**(不强调状态色,以保持视觉一致),最小宽 34pt 右对齐 |
| 行内 spacing | 8pt(tile / bar / 百分比 三段之间) |

进度 = `fiveHour.remainingPercent / 100`。状态色按 设计风格 §4.3 覆盖(<20% 时 bar + 文字一起变橙,= 0 变红)。

### 2.3 备选变体(Settings 提供切换)

prototype 还画了另外 7 个变体,Settings → Floating HUD → Style 下拉里全部提供:

| Variant | 名称 | 适用场景 |
|---|---|---|
| 1(默认) | Two-row pill · 两行胶囊 | 标准 |
| 2 | Stacked numbers · 大数字 | 不要 bar,只看百分比 |
| 3 | Dual mini rings · 双小环 | 视觉化偏好者 |
| 4 | Ultra-thin · 极简 | 屏幕空间紧张 |
| 5 | Single service · 单服务 | 只关心 Claude / 只关心 Codex |
| 6 | Horizontal slim · 横向窄条 | 顶部 / 底部窄边贴边 |
| 7 | 5h + Week verbose · 详细模式 | 同时看 5h 和周 |
| 8 | No chrome · 无背景纯文字 | 极致简约,文字带阴影直接铺桌面 |

实现:每个 variant 一个 `View`,在 `FloatingContentView` 里按 settings 切换。

### 2.4 交互

| 触发 | 行为 |
|---|---|
| 整个 HUD 区域拖动 | 移动位置,松手时持久化到 SettingsStore |
| 拖至屏幕边 20pt 内 | 吸附到边缘 |
| 鼠标进入 / 离开 | 透明度根据 Settings → Opacity when idle 切换(idle 0.7 → hover 1.0,150ms ease) |
| 点击单行 | 暂不响应(规划中:跳转主窗口对应服务 scope) |
| 右键 | 显示原生菜单:`Hide HUD · 隐藏` / `Settings · 设置` |

### 2.5 浅深色

- **不写**手动 background;`NSVisualEffectView(.hudWindow)` 在两种 appearance 下自动给出 prototype 的两套 alpha + blur。
- 文字颜色:标签 `.secondary`、百分比服务色(浅深色双套,见 设计风格 §4.2 / §12.2)。
- **不**加 strokeBorder。当前 `Floating/FloatingContentView.swift:35-38` 的 `strokeBorder(opacity 0.08)` 应去掉。

### 2.6 数据来源对照

| UI 字段 | 数据源 |
|---|---|
| 行 1 百分比 + bar | `appState.codexQuota?.fiveHour.remainingPercent` |
| 行 2 百分比 + bar | `appState.claudeQuota?.fiveHour.remainingPercent` |
| 显示 / 隐藏 | `settings.floatingEnabled` |
| 哪些服务 | `settings.floatingShowCodex / floatingShowClaude` |
| Variant | `settings.floatingVariant`(新字段,enum,默认 `.twoRowPill`) |
| 位置 | `settings.floatingPosition`(CGPoint,持久化) |
| Idle 透明度 | `settings.floatingIdleOpacity`(Double,默认 0.7) |

## 3. Statistics 主窗口

### 3.1 整体规格(默认视图:Overview)

| 项 | 值 |
|---|---|
| 宽 / 高 | 默认 1080pt × 680pt,最小 800pt × 520pt,可调 |
| 圆角 | 12pt(系统 Window) |
| 工具栏 | 系统 toolbar |
| 布局 | Sidebar(200pt) + Main canvas |

### 3.2 Toolbar

- **左**:`Statistics` 13pt .semibold + `用量统计` 11pt secondary,水平间距 10pt。
- **右**:SegmentedControl,9 个段:
  `Today / Yesterday / Week / Month / Year / 7d / 30d / All / Custom`
  对应 03 词表:`今天 / 昨天 / 本周 / 本月 / 本年 / 7 天 / 30 天 / 全部 / 自定义`(中文不显示在控件里,而是在 Sidebar Range 组里显示)。
  - 容器:2pt padding,圆角 7pt,bg `rgba(0,0,0,0.06)`(浅)/ `rgba(255,255,255,0.08)`(深)
  - 段:`3pt 上下 / 10pt 左右`,高 22pt,圆角 5pt
  - active:浅色 `#fff` + 微阴影 `0 0.5 2 rgba(0,0,0,0.18)`;深色 `rgba(120,120,128,0.55)` + 同阴影;字体 .semibold,letterSpacing -0.05
  - **SwiftUI 推荐**:直接用 `Picker(.segmented)`。如果系统外观不够还原,再自绘。

### 3.3 Sidebar(200pt 宽)

- 整体:`.sidebar` material,padding `14pt 上下 / 12pt 左右`,内部 VStack gap 16pt。
- 右侧 0.5pt separator(分隔 sidebar 与 main)。

**三个 Group**,每个 group:

- **Group title**:10pt .semibold letterSpacing 0.4 tertiary 大写,padding `0pt 6pt 4pt 6pt`,格式 `RANGE · 时间范围`。
- **Items**(VStack,gap 1pt):每行 `5pt 上下 / 8pt 左右`,圆角 6pt,12pt .medium,**active 行**:accent 背景 + 白字。

| Group | Items(EN / CN) | 说明 |
|---|---|---|
| Range · 时间范围 | Today / Yesterday / This week / This month / This year / Last 7 days / Last 30 days / All time / Custom… | 与 toolbar segmented 同步;每行带一个 13pt 日历 icon |
| Service · 服务 | All / Codex / Claude Code | All 用 8pt 空占位;Codex / Claude 用 8×8 squircle(radius 2,服务色)作为前缀色块 |
| View · 视图 | Overview / Timeline / Breakdown | 各带 13pt icon:`rectangle.split.2x2` / `chart.line.uptrend.xyaxis` / `list.bullet` |

每行**右侧**有 10.5pt tertiary 的中文 caption(active 行 opacity 0.75,非 active 0.5)。

### 3.4 Main canvas — Overview(主视图)

整体 padding 20pt,VStack gap 12pt(KPI 行下 18pt)。

#### 3.4.1 顶部 KPI 行(4 卡)

Grid `repeat(4, 1fr)`,gap 12pt。每卡:

| 项 | 值 |
|---|---|
| 圆角 | 10pt |
| 背景 | 浅 `#fff` / 深 `rgba(60,60,63,0.4)`(见 设计风格 §12.3) |
| 内描边 | 0.5pt(见 设计风格 §12.3) |
| Padding | 11pt 上下 / 14pt 左右 |

内结构(VStack):

1. **Label 行**(HStack,baseline,gap 4):
   - 可选 6×6 squircle radius 1.5 服务色块(仅服务卡有)
   - `Label` 11pt secondary
   - `Cn` 10pt tertiary(中文 caption)
2. **Value 行**(HStack,baseline,gap 8,marginTop 4):
   - 主值 22pt .semibold kerning -0.5 tabular(服务卡用服务色,合计卡用 primary)
   - Delta 11pt .semibold tabular,红涨绿跌,格式 `↑ 12.4%`(red) / `↓ 2.3%`(green),**裸文字无 chip**

四张卡内容:

| Card | Label / CN | Value | Delta |
|---|---|---|---|
| 1 | `Total tokens` / `总令牌` | `48.2M` | `↑ 12.4%` |
| 2 | `Total spend` / `总花费` | `$1,284.50` | `↑ 8.1%` |

| 4 | `Claude Code` / `Anthropic`(桃色块) | `$872.20` 服务色 | `↑ 18.6%` |

### 3.5 Main canvas — Timeline(备选视图)

切换到 sidebar → View → Timeline 时显示。

- **Tokens · hourly panel**:容器同 Overview Panel,内部 `AreaLineChart` 高 220pt。
  - 两条 smooth line(SVG path),Codex 深 + Claude 桃,strokeWidth 1.6pt
  - 区域填充 gradient:服务色 0.4 → 0 透明度
  - 4 条水平 gridline(20/40/60/80),0.2pt,浅 `rgba(0,0,0,0.05)` / 深 `rgba(255,255,255,0.06)`
