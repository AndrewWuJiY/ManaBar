# cc-bar 产品需求

> 描述当前已交付版本的功能范围与边界。新需求迭代时以本文为准；与代码不一致以代码为准并同步回这里。

---

## 0. 产品定位

- **形态**：常驻菜单栏的小工具（Accessory App，不占 Dock）。
- **平台**：macOS 14 Sonoma+（依赖 SwiftUI `MenuBarExtra(.window)`、`@Observable`、`SMAppService`）。
- **设计**：以 macOS HIG 为底座，关键信息层（百分比、进度、KPI、HUD）按设计稿落地自绘组件；非关键层（Picker / Toggle / DatePicker / TabView / Material）全部用系统默认。
- **一句话价值**：装上它 → 不打开 CLI 也能随时看到 Codex / Claude Code 还剩多少 5 小时与周窗口额度、今天和本周花了多少钱。

---

## 1. 账号自动识别

- 启动时扫描本机已有的 Codex / Claude Code 凭据：
  - Codex：`~/.codex/auth.json`（OAuth 模式，解析 JWT 拿邮箱 / plan / accountId）。
  - Claude：先读 `~/.claude/.credentials.json`；不存在则读 macOS Keychain 通用密码项 `Claude Code-credentials`（service name）。邮箱兜底从 `~/.claude.json` 顶层 `oauthAccount.emailAddress` 读取。
- 不做 App 内登录流程。前提：用户已经在终端跑过 `codex login` / `claude` 登录。
- 用户可在「设置 → 账号」勾选要显示的服务；关闭后该服务在菜单栏、Popover、悬浮窗、统计中均不出现，也不查询其 API。
- 首次启动若不存在 `~/.claude/.credentials.json`，且尚未提示过，会先弹一次说明性 Alert（"接下来 macOS 会弹 Keychain 授权窗口，请选择始终允许"），再触发系统 Keychain 弹窗。

## 2. 菜单栏标签

- 标签是模板图像（template NSImage），自适应深浅菜单栏。
- 顺序固定：**Codex 在前，Claude 在后**。
- 每段为 `<服务 logo>  <百分比>`：
  - Logo 用项目内置的 `codex.svg` / `claude.svg`（不是 SF Symbol，保留服务的颜色辨识度——菜单栏会做模板化变灰）。
  - 百分比用 `monospacedDigit`。
- 设置项：
  - 显示哪个服务（Codex / Claude，可分别开关）。
  - 显示哪个额度窗口：5 小时 / 周窗口 / 两者（`5h/1w` 斜杠拼接）。
- 都关闭时仍保留一个占位 logo，避免菜单栏完全消失。

---

## 3. Popover 总览

- 点击菜单栏图标弹出宽 340pt 的窗口。
- 顺序：Codex → Claude（每个服务一个 block，用 Divider 分隔）。
- Header 一行：标题「Usage / 用量」+ 副标题（最近一次刷新相对时间）+ 状态点（live / stale / offline）+ 三个一级图标按钮（刷新 / 统计 / 设置）+ ⋯ kebab 菜单（退出）。
- 每个 block 字段：
  - 服务 tile + 服务名 + 副标题（Provider · Plan）。
  - 5 小时窗口：38pt 剩余百分比 + 进度条 + 重置倒计时 + 「今日 cost」+「本周 cost」。
  - 周窗口：缩小版进度条 + 百分比 + 重置时间。
  - Claude block 在周窗口下方追加 Claude Design 周额度行，样式与周窗口行一致；旧缓存或接口未返回该字段时显示占位。
  - 失败时下方两行内短文案提示。
- 关闭两个服务时显示"未启用任何服务 / 到「设置 → 账号」开启"占位。
- 状态点阈值：以当前额度刷新间隔为基准，age ≤ 1.5× 间隔为 live，≤ 3× 为 stale，超过为 offline。

---

## 4. 桌面悬浮窗（Floating HUD）

- 设置里可勾选「显示悬浮窗」。
- 默认两行 pill：
  - 第一行：Codex 5 小时剩余百分比（18pt tile + 进度条 + 大字百分比）。
  - 第二行：Claude 5 小时剩余百分比。
- 行可分别开关，结果只显示一行 / 全部隐藏。
- 样式：14pt 圆角 + `.popover` material + 0.5pt hairline 描边；无标题栏；不抢焦点；可拖动；始终置顶。
- 位置记忆：拖动后写回 UserDefaults；屏幕外重启时回落到默认右上角。
- 拖到距屏幕边缘 20pt 以内自动吸附到该边。

## 5. 自动后台刷新

- 后台两条独立循环：
  - 额度 API 循环：默认 5 分钟，可选「关闭 / 30s / 1m / 5m / 10m」。
  - 本地 JSONL 扫描循环：默认 30 秒，可选「15s / 30s / 1m / 5m」。
- 端点：
  - Codex：`GET https://chatgpt.com/backend-api/wham/usage`，带 `ChatGPT-Account-Id` 头。
  - Claude：`GET https://api.anthropic.com/api/oauth/usage`，带 `anthropic-beta: oauth-2025-04-20`；Claude Design 周额度来自响应中的 `seven_day_omelette` bucket。
- 限流策略：
  - 同一服务两次成功之间至少 60 秒，周期性触发若间隔不足直接跳过。
  - 收到 429 后该服务进入 10 分钟退避，期间所有请求短路。
  - 手动「立即刷新」可绕过 60s 节流但仍遵守 429 退避。
- Claude API 失败且**完全没有可展示缓存**时，手动刷新会触发一次 Claude CLI 兜底（启动 `claude` 进程，向其 stdin 写入 `/usage`，解析输出），CLI 兜底也有独立 10 分钟冷却。
- Token 续期：Codex / Claude 双方均在 access_token 距离过期 5 分钟内时用 refresh_token 自动续期，并把新 token 写回原存储位置（文件或 Keychain）。
- 启动行为：先读本地额度缓存让 UI 立刻有数；首次刷新由调度器节奏触发；网络失败保留已有快照，仅以小字提示。

## 6. 本地用量统计

- 数据源：纯本地 JSONL，**额度 API 不返回历史数据**。
  - Claude：`~/.claude/projects/**/*.jsonl`，解析 `type=assistant` 行。同一 `message.id` 跨文件全局去重（sidechain / subagent 会重复引用同一条），seen 集合上限 20000。
  - Codex：`~/.codex/sessions/**/*.jsonl` + `~/.codex/archived_sessions/**/*.jsonl`，按 `turn_context` 提供的当前 model 给随后的 `event_msg/token_count` 打标签；token 用 `payload.info.last_token_usage`，billable input = `input_tokens − cached_input_tokens`。
- 增量逻辑：每个文件按 `mtime + offset` 维护 watermark，未变化直接跳过；文件被截断时 offset 归零重扫，去重由全局 seen 兜底。
- 聚合：按 `(day, app, model)` 三元组在内存中累加；持久化到 `usage-rollup.json`。
- 价格表：在 `Core/Usage/Pricing.swift` 中硬编码，对齐 cc-switch / CodexBar 实测值（claude opus / sonnet / haiku 4.x、gpt-5 至 gpt-5.6 含 codex 变体）。命中不到的模型 cost 计 0，token 仍记录。模型名归一化：剥 `openai/` 前缀、`@YYYYMMDD` 或 `-YYYY-MM-DD` 日期后缀。

## 7. 主窗口 — 统计

- 主窗口为普通 macOS 窗口，最小 800×520，含顶部 TabView：统计 / 设置。
- 入口：Popover 的统计 / 设置图标、菜单栏的 ⌘1 / ⌘, 命令、⌘R 触发立即刷新。
- 统计 tab：
  - 侧栏：服务过滤（All / Codex / Claude Code），View 切换占位（Overview 可用，Timeline / Breakdown 灰显）。
  - 顶部 segmented Picker：Today / Yesterday / Week / Month / Year / 7d / 30d / All / Custom。
  - 自定义：两个 DatePicker（仅在 Custom 时出现）。
  - KPI 一排 4 张：总 tokens / 总花费 / Codex 花费 / Claude 花费；每张带相对上一同长度区间的 delta 百分比。
  - 日趋势：堆叠柱状图（Codex vs Claude），用 SwiftUI Charts。
  - 服务占比：水平条形 + KPI 文本。
  - 当前额度：圆环展示 Codex / Claude 的 5h / 1w 剩余百分比，颜色按状态色逻辑。
  - 按模型明细：分服务列出 model · 调用次数估算 · tokens · cost。

## 8. 设置

| 分类 | 项 |
|---|---|
| 账号 | 显示 Codex、显示 Claude（自动检测的邮箱 / plan 显示在副标题） |
| 菜单栏 | 显示 Codex、显示 Claude、额度周期（5h / 周 / 两者） |
| 桌面悬浮窗 | 显示悬浮窗、显示 Codex 行、显示 Claude 行 |
| 刷新 | 额度刷新间隔、日志扫描间隔、重置时间显示（剩余时长 / 具体时间）、上次刷新（只读） |
| 通用 | 语言（跟随系统 / 中文 / English）、隐私模式（Popover 中隐藏主账号邮箱、Codex 副账号名称）、开机自动启动（SMAppService）、版本（只读） |

切换额度 / 日志间隔后会立即同步到 Scheduler，无需重启。

## 9. 首次启动引导（Onboarding）

- 首次启动检测到 `SettingsStore.didCompleteOnboarding == false` 时弹出 4 步窗口（620×520，`.regularMaterial`）：
  1. Welcome：标题 + 简介。
  2. Detect accounts：展示自动识别的 Codex / Claude 账号、邮箱、plan，未识别时给出"请到终端登录"提示。
  3. Configure：菜单栏窗口 / 悬浮窗开关等关键设置一次性勾选。
  4. Ready：完成，按钮可直接进入"统计"或关闭。
- 完成后写入 `didCompleteOnboarding=true`，后续启动不再展示。

## 10. 双语

- 全局 `tr(en, zh)` 选词，由 `SettingsStore.resolvedLanguage` 决定渲染。
- `跟随系统` 看 `Locale.preferredLanguages.first`，以 `zh` 开头判定为中文（兼容简繁）。

## 12. 一期明确不做

- 不做 App 内 OAuth 登录。
- 不做浏览器 Cookie 自动导入。
- 不做 WebView 抓 dashboard DOM。
- 不做 Gemini / Copilot / 国产 Token Plan。
- 不上 Mac App Store（沙盒会拦住 `~/.claude` / `~/.codex` 读取）。
- 不做组织级 Admin API。
- 不做额度告警通知。

`claude /usage` 的 CLI 兜底当时被列为「一期不做」，实际作为最后一道兜底已经做了；只在 Claude API 失败且无可展示缓存的手动刷新场景启用，不替代主路径。

## 13. 风险 / 待持续关注

1. **非公开接口稳定性**：`wham/usage` / `oauth/usage` 字段可能随时调整，解析器需要保持容错。
2. **Keychain 弹窗**：首次读 `Claude Code-credentials` 系统会弹一次授权窗，引导文案已加。
3. **价格表漂移**：当前对齐 cc-switch / CodexBar 2026 上半年价位；模型迭代后需要同步。
4. **JSONL 量级**：长期使用后 `~/.claude/projects` 可达数百 MB；增量 watermark + seen ID 上限保证启动 / 扫描时间可控，但 seen 集合上限 20000 时若历史窗口超过该长度会有边缘重复风险。
5. **签名与公证**：发布前需 Apple Developer 证书；当前开发期未签名，README 标注「右键打开」。
