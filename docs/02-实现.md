## 0. 参考项目源码位置（本机）

实施 / 排错 / 对照行为时，**优先**到下面这三份项目的代码里找答案，再决定我们的实现：

| 项目 | 本机路径 | 栈 | 重点参考 |
|---|---|---|---|
| **CodexBar** | `~/Code/CodexBar` | Swift / SwiftPM | 整个 macOS 菜单栏 App 结构；Claude 的 `OAuth API -> CLI PTY -> Web Cookie` fallback pipeline；provider 图标资源（`Sources/CodexBar/Resources/ProviderIcon-*.svg`） |
| **cc-switch** | `~/Code/cc-switch` | Rust / Tauri | tray 节流 + on-hover lazy refresh 思路（`src-tauri/src/tray.rs`）；Codex / Claude 订阅额度都直接调官方隐藏 API，主要靠低频触发 + 内存缓存 |
| **cockpit-tools** | `~/Code/cockpit-tools` | Rust / Tauri | Codex 配额查询（`crates/cockpit-core/src/modules/codex_quota.rs`）走 `wham/usage`；错误快照、原始响应保留、缓存结构可借鉴；**不作为 Claude Code 网络层参考** |

> 结论：Claude Code 的订阅窗口官方隐藏接口仍以 `GET https://api.anthropic.com/api/oauth/usage` 为主。CodexBar 额外做了 CLI PTY `/usage` 和 Web Cookie fallback，但 Web Cookie 需要额外权限且风险更高。本项目采用低风险折中：默认只打 OAuth API；失败时保留缓存；仅在「手动刷新 + 没有可展示缓存」时尝试 Claude CLI `/usage` 兜底；不做浏览器 Cookie 自动导入。

## 1. 技术栈

| 层 | 选择 | 理由 |
|---|---|---|
| 语言 | Swift 6 | 原生、严格并发、未来友好 |
| 构建 | **Xcode 工程**（`ccbar.xcodeproj`） | 标准 macOS App 工程，Xcode 打开 ⌘R 直接跑；签名/公证/Assets 都走 Xcode 默认流程，最省事 |
| UI | SwiftUI 为主，AppKit 兜底 | 「保持默认 macOS 风格」最省事的选择 |
| 最低系统 | macOS 14 Sonoma | 才有 `MenuBarExtra`、`WKWebsiteDataStore(forIdentifier:)` 等新 API |
| 存储 | 轻量 JSON + UserDefaults（一期不上 SQLite） | 数据量不大，简单为先 |
| 网络 | `URLSession` async/await | 标准库 |
| 通知 | 不做（一期跳过） | |
| 自动更新 | 一期手动发布，二期再接 Sparkle | |

**为什么不选 Tauri**：参考的 cc-switch / cockpit-tools 是 Tauri，对于「想要严格遵循 macOS 最新设计规范」反而是劣势——WKWebView 渲染的 UI 怎么调都有「网页味」。原生 SwiftUI 一行不写就是 macOS 风格。

## 3. 关键流程

### 3.1 凭据读取

**Codex**：
1. 读 `~/.codex/auth.json`（环境变量 `CODEX_HOME` 优先）。
2. JSON 里取 `tokens.access_token` / `refresh_token`。
3. 解 JWT 的 `exp` 字段，留 5 分钟 skew。
4. 过期就 `POST https://auth.openai.com/oauth/token` 续期（grant_type=refresh_token, client_id 是 Codex CLI 的公开 ID `app_EMoamEEZ73f0CkXaXp7hrann`），刷新后**原子写回 `auth.json`**（先写 .tmp 再 rename）。
5. 从 `id_token` 里解 `chatgpt_plan_type` / `account_id` 当作账号信息。

**Claude**：
1. 优先读 `~/.claude/.credentials.json`，取 `claudeAiOauth.accessToken` + `expiresAt`。
2. 文件不存在或为空 → 走 `security find-generic-password -s "Claude Code-credentials" -w"` 命令读 Keychain（避免直接 SecItemCopyMatching 触发弹窗的复杂策略，一期简单粗暴）。
3. 一期**不做** Claude token 自动刷新（Anthropic OAuth 刷新流程复杂，过期就提示用户重登）。

### 3.2 额度查询

**接口形态**：

| 应用 | 请求 | 关键 Header |
|---|---|---|
| Codex | `GET https://chatgpt.com/backend-api/wham/usage` | `Authorization: Bearer ...`、`ChatGPT-Account-Id: <从 JWT 解>`、`User-Agent: codex-cli` |
| Claude | `GET https://api.anthropic.com/api/oauth/usage` | `Authorization: Bearer ...`、`anthropic-beta: oauth-2025-04-20`、`User-Agent: claude-code/<ver>`（伪装 CLI，**对齐 CodexBar**，可降低被限流概率） |

**响应归一化**为统一结构：

```swift
struct QuotaSnapshot: Codable {
    let app: AppKind            // .codex / .claude
    let fiveHour: QuotaWindow?
    let weekly: QuotaWindow?
    let weeklyOpus: QuotaWindow?    // 仅 Claude
    let weeklySonnet: QuotaWindow?  // 仅 Claude
    let planType: String?
    let fetchedAt: Date
}
struct QuotaWindow: Codable {
    let usedPercent: Double     // 0~100，接口原始语义：已使用百分比
    var remainingPercent: Double { max(0, min(100, 100 - usedPercent)) }
    let resetsAt: Date?
}
```

**调用策略（对齐 cc-switch / CodexBar 的低风险做法，避免触发 429）**：

| 项 | 取值 | 出处 / 理由 |
|---|---|---|
| 首次拉取 | App 启动完成凭据后立即拉一次 | 让 UI 第一次出现就有数 |
| 首次拉取 | App 启动只读 `quota-cache.json` + 凭据，不立刻打网络 | 避免 Xcode 热更新 / App 重启时频繁调用 Claude 接口 |
| 周期拉取 | M3 默认 **2 分钟**；正式默认可改回 5 分钟（可在设置改 1~30 分钟） | 既保留自动更新，又比代码热更新/Popover 打开触发更安全 |
| 触发拉取 | ① 周期 tick ②「刷新」按钮 ③ 设置里手动「立即刷新」 | Popover 打开只展示当前状态，不主动请求 |
| 节流 | 同一家上次成功 < 60 秒内的周期触发**跳过**，手动刷新可绕过普通节流 | 防止后台重复请求；手动刷新仍遵守 429 退避 |
| 429 退避 | 收到 `429` 后，该家**至少 10 分钟**内不再发请求；保留上一次的缓存继续显示 | cc-switch 没有显式退避（靠 lazy 调用频率本来就低），我们因为有后台周期，必须显式退避避免雪崩 |
| 401 / 403 | 标记凭据失效，停止周期拉取，UI 提示「请重新登录」 | |
| 网络/其它错误 | 不清空缓存；下一次 tick 正常重试；UI 底部用小字标错误 | 「降级不崩」 |
| Claude fallback | 仅在「用户手动刷新 + API 失败 + 当前没有 Claude 快照」时运行 `claude /usage`，解析屏幕中的窗口百分比 | 对齐 CodexBar 的 CLI 兜底，但避免后台自动跑 CLI |

**持久化缓存（关键改动）**：

- 每次成功响应后写一份到磁盘：`~/Library/Application Support/com.nanvon.ccbar/quota-cache.json`
  ```jsonc
  {
    "codex":  { /* QuotaSnapshot JSON */ },
    "claude": { /* QuotaSnapshot JSON */ }
  }
  ```
- App 启动顺序：① 同步读这份缓存灌进 AppState（**用户立刻看到上次的数字**，不再是空白）→ ② 拉凭据 → ③ 等周期 tick 或用户点击刷新再请求网络。
- 请求失败时**不动磁盘缓存也不动内存快照**，UI 继续显示旧数字 + 底部小字「上次更新 5 分钟前 · 当前请求失败」。
- 缓存格式向前兼容：解码失败就当无缓存，不抛错。

**M3 已落地的安全刷新约束**：

- `Scheduler` 默认 120s 周期；App 启动不立刻请求网络。
- `AppState` 对 Codex / Claude 分别维护 `lastSuccessAt`、`lastAttemptAt`、`backoffUntil`、`lastError`、`inFlight`。
- 成功快照写入 `~/Library/Application Support/com.nanvon.ccbar/quota-cache.json`；启动先读缓存展示。
- 请求失败不清空内存快照；429 设置 10 分钟退避。
- Claude quota 请求补 `User-Agent: claude-code/2.1.0`，并仅在手动刷新且没有可展示缓存时尝试 CLI `/usage` 兜底。

### 3.3 本地 JSONL 增量扫描

**Claude**：
- 路径：`~/.claude/projects/**/*.jsonl`（含子目录的 `subagents/*.jsonl`）。
- 每行 JSON，只关心 `type == "assistant"`。
- 取 `message.id` 去重（保留 `stop_reason != nil` 的那条），`message.usage` 拿 token 数。
- 时间用行内 `timestamp`，模型用 `message.model`。
- 注意：Claude 的 `input_tokens` **不含** cache_read。

**Codex**：
- 路径：`~/.codex/sessions/YYYY/MM/DD/*.jsonl` + `~/.codex/archived_sessions/*.jsonl`。
- 关心 `event_msg.payload.type == "token_count"`（**累计值**，要算 delta）。
- `turn_context` 拿模型（剥掉 `openai/` 前缀和日期后缀）。
- 注意：Codex 的 `input_tokens` **含** cache_read，算钱时要扣。

**增量机制**：
- 维护一份缓存文件 `~/Library/Caches/<bundleId>/scan-state.json`：
  ```jsonc
  {
    "claude": { "<file path>": { "mtime": 12345, "offset": 678 } },
    "codex":  { "<file path>": { "mtime": 12345, "offset": 678, "lastCumulative": {...} } }
  }
  ```
- 每次扫描：列文件 → 比 `mtime` → 只读 `offset` 之后的字节 → 更新缓存。
- 解析结果累加到一张**按 (day, app, model) 分桶**的内存表上，再 dump 到 `~/Library/Application Support/<bundleId>/usage-rollup.json`。

**价格表**：
- 硬编码在 `Pricing.swift`，参考 cc-switch 的 `seed_model_pricing` 抄一份（GPT-5、Sonnet/Opus 4.x 等）。
- 金额用 `Decimal`（Swift 原生），不用 `Double`。
- 二期可考虑后台拉 `https://models.dev/api.json` 覆盖。

### 3.4 后台调度

一个 `@MainActor` 的 `Scheduler` 持有 `AppState` 弱引用，对每家**独立**维护：

- `lastSuccessAt: Date?` —— 上次成功时间，用于「< 60s 跳过」的节流
- `nextEarliestRefreshAt: Date?` —— 退避截止时间（429 时设为 `now + 10min`）
- `task: Task<Void, Never>?` —— 周期循环

刷新触发点统一走 `Scheduler.refresh(app:reason:)`：
- `reason = .periodic`：来自后台定时器
- `reason = .userInitiated`：来自刷新按钮 / 设置「立即刷新」—— **忽略普通 60s 节流**，但仍然遵守退避（429 期间也不放行，避免用户狂点）

周期：
- 额度 API：**默认 300s**（5 分钟，可在设置改）
- JSONL 扫描：默认 30s（M5 引入，独立于额度，本地文件不存在限流问题）

每次 tick 触发对应 client → 写 `AppState`（`@Observable`）→ 成功时同步写 `quota-cache.json`。UI 通过 SwiftUI 的 Observation 自动刷新。

### 3.5 菜单栏渲染

- 用 `MenuBarExtra`（SwiftUI 14+）最省事；如果发现需要更精细控制（比如标题文字动态变化）再换 `NSStatusItem`。
- 标题文字由 `MenuBarLabelFormatter` 根据当前设置 + `AppState` 拼接：
  - 没有数据 → 只显示图标
  - 只勾 Codex 5h → `34%`
  - 同时勾两个 → `C 34% · K 60%`（缩写和分隔符待 UI 阶段定）
- 图标用 SF Symbol `chart.bar.fill` 或类似的，模板模式自动适配深浅菜单栏。

### 3.6 弹出面板

- `MenuBarExtra` 的 `.style(.window)`，里面用纯 SwiftUI 列表。
- 顺序写死：Codex 区块在上，Claude 区块在下。
- 每个区块用系统的 `GroupBox` + `ProgressView` 渲染进度条，完全跟随系统配色。
- 底部 `Divider()` 后接「刷新 / 设置 / 退出」一行 Button（顺序固定，无「详细」入口；详见 [04-界面布局.md](04-界面布局.md) §2）。

### 3.7 桌面悬浮窗

- 新开一个 `NSWindow`（SwiftUI 的 `Window` scene 不够灵活）：
  - `styleMask = [.borderless]`
  - `level = .floating`（或可选 `.statusBar`）
  - `isOpaque = false`、`backgroundColor = .clear`
  - 内容是 SwiftUI 的两行文字（用 `NSHostingView` 包）
- 拖动：重写 `mouseDown`/`mouseDragged` 或用 `.gesture(DragGesture())` + `window.setFrameOrigin`。
- 位置存 UserDefaults。
- 不抢焦点：`canBecomeKey` / `canBecomeMain` 返回 false。

### 3.8 主窗口（统计 + 设置）

- 一个普通的 `Window` scene（**不是** macOS 标准 `Settings { }` scene），系统默认外观。
- 顶部 `TabView` 两个 tab：**统计 / 设置**，默认停在「统计」tab。
- **统计 tab**：
  - 顶部时间范围选择器（今天 / 本周 / 本月 / 7 天 / 30 天 / 全部 / 自定义）
  - 自定义时显示两个 `DatePicker`
  - 汇总卡片：Codex 一张、Claude 一张（分开统计，不合计）
  - 按模型拆分列表（每个应用一组）
  - 一期不画图表
- **设置 tab**：5 个分组（账号 / 菜单栏 / 悬浮窗 / 刷新 / 通用），所有项绑定 `@AppStorage`，详见 [04-界面布局.md](04-界面布局.md) §3.2。
- 入口：Popover 底部「设置」按钮 → 打开主窗口（默认统计 tab，但是按钮触发的可以指定切到设置 tab）。
- 数据来源：`UsageAggregator` 按时间范围对内存桶做求和。

> **当前现状**：仓库里的 `Settings/SettingsRootView.swift` 还挂在 `Settings { }` scene 上。M6 阶段会迁到这里描述的 `Window` scene + TabView 结构。

## 4. 关键技术点速查

| 主题 | 用什么 | 备注 |
|---|---|---|
| 菜单栏 | `MenuBarExtra(...) { ... } label: { ... }` | macOS 13+ |
| 隐藏 Dock | `Info.plist` 加 `LSUIElement = true` | 必须 |
| 读 Keychain | `Process` 调 `security find-generic-password` | 简单粗暴，一期够用 |
| JWT 解析 | 手写 `base64decode(jwt.split(".")[1])` | 不用引第三方库 |
| 原子写文件 | 写 `path.tmp` → `FileHandle.synchronize` → `FileManager.moveItem` | 防半文件 |
| 后台并发 | `actor` + `Task { ... }`，UI 更新前 `await MainActor.run` | Swift 6 严格并发 |
| Decimal 算钱 | Swift `Decimal` / `Foundation.Decimal` | 不用 Double |
| 开机自启 | `SMAppService.mainApp.register()` | macOS 13+ |
