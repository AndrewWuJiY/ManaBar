# cc-bar 技术实现

> 对应 [产品需求.md](产品需求.md)。描述当前代码的架构、模块边界与关键流程。新增功能时以本文作为入门地图，与代码不一致以代码为准。
> 历史参考（外部项目对照、设计稿、初始里程碑）见 [历史参考/](历史参考/)。

---

## 1. 技术栈

| 层 | 选择 | 备注 |
|---|---|---|
| 语言 | Swift 6 | 严格并发 |
| 工程 | Xcode `ccbar.xcodeproj` | 单 target，标准 macOS App |
| UI | SwiftUI 主、AppKit 辅 | 菜单栏 label / 悬浮窗 panel 走 AppKit |
| 最低系统 | macOS 14 Sonoma+ | 依赖 `MenuBarExtra(.window)`、`@Observable`、`SMAppService` |
| 存储 | UserDefaults + JSON 文件 | 不引入 SQLite |
| 网络 | `URLSession` async/await | 标准库，无第三方依赖 |
| 图表 | `Charts`（SwiftUI） | 用于统计 tab 日趋势 |

整个工程零第三方依赖；钥匙串通过命令行 `security` 访问，避免 Keychain Services API 的多次授权弹窗。

## 3. 应用入口与窗口组合

`CCBarApp.swift` 组装三个 Scene：

1. `MenuBarExtra(.window)`：label 是自绘 NSImage 包装；content 是 `PopoverRootView`。`.task` 内执行 `appState.bootstrap()` 并把 `FloatingPanelController` 与 AppState 绑定，再根据设置同步显示。
2. `Window("CCBar", id: "main")`：主窗口，内部 `MainWindowRootView` 用 TabView 切「统计 / 设置」。
3. `Window("Welcome", id: "onboarding")`：引导窗口，`fixedSize` 禁缩放。

`AppCommands` 通过 `CommandGroup` 注入全局快捷键：

- `⌘,` → 打开主窗口并切到设置 tab。
- `⌘1` → 打开主窗口并切到统计 tab。
- `⌘R` → 调用 `appState.refreshNow()`。

`AppDelegate` 只做一件事：`NSApp.setActivationPolicy(.accessory)` 让进程不出现在 Dock。

## 4. 全局状态：`AppState`

`@Observable @MainActor`，单例方式由 `CCBarApp` 持有并通过 `.environment` 注入。核心字段：

- 凭据：`codexAccount` / `claudeAccount`，加载错误 `codexError` / `claudeError`。
- 额度：`codexQuota` / `claudeQuota`（`QuotaSnapshot`），来源 `codexQuotaSource` / `claudeQuotaSource`（`api` / `cache` / `cliFallback`）。
- 刷新状态：`QuotaRefreshState`（`lastSuccessAt`、`lastAttemptAt`、`backoffUntil`、`lastError`、`inFlight`、`source`）。
- 今日花费：`codexTodayCost` / `claudeTodayCost`，由 `UsageService.publishTotals()` 写入。
- UI 协调：`mainTab`（统计 / 设置）、`shouldShowOnboarding`。
- 子系统：`usageService`、`scheduler`、`quotaCache` 内存副本、Claude CLI 兜底冷却 `claudeFallbackBackoffUntil`。

关键方法：

- `bootstrap()`：只执行一次；按顺序加载 quota 缓存 → `UsageService.bootstrap` → 读 Codex 凭据 → 必要时弹 Keychain 引导 → 读 Claude 凭据 → 决定是否展示引导 → 启动 Scheduler → 触发首次 JSONL 扫描（异步不阻塞）。
- `refreshNow()`：用户手动刷新；缺凭据则重读，触发一次额度刷新和一次扫描。
- `refreshQuotas(reason:)`：分别调度 Codex 与 Claude 额度刷新，写日志摘要。
- `applySettingsChange()`：设置项变化后立即把新间隔灌进 Scheduler。

刷新前置校验在 `beginCodexRefresh` / `beginClaudeRefresh` 中：检查 `inFlight`、`backoffUntil`、周期性触发的 60s 最小间隔。成功后 `storeXxx` 写入快照 + 持久化；失败走 `markXxxFailure`，遇 429 设置 10 分钟退避。

## 5. 凭据读取

### 5.1 Codex

`CodexAuth.load()` 读 `~/.codex/auth.json`，解析 `tokens.access_token` / `refresh_token`，再用 `JWT.decode` 解 `id_token` 的 payload 拿邮箱、`chatgpt_plan_type`、`account_id`。

`CodexTokenRefresher.ensureFreshAccessToken`：当 JWT `exp` 剩余 < 300s 时 POST 到 OpenAI OAuth token 端点（grant_type=refresh_token，client_id 用 Codex CLI 公开 ID），成功后原子写回 `auth.json`。

### 5.2 Claude

`ClaudeAuth.load()` 优先读 `~/.claude/.credentials.json`；不存在再调 `security find-generic-password -s "Claude Code-credentials" -w` 读 Keychain。邮箱兜底来自 `~/.claude.json` 顶层 `oauthAccount.emailAddress`。

`ClaudeTokenRefresher.ensureFreshAccessToken`：

- 端点 `https://platform.claude.com/v1/oauth/token`，client_id 用 Claude Code CLI 的公开 ID。
- 续期成功后按原存储位置（`.file` / `.keychain`）回写。Keychain 回写先用 `security find-generic-password` 读出原 JSON，merge token 字段后用 `security add-generic-password -U -s <service> -a $USER -w <json>` 整体覆盖，保留 `subscriptionType` 等附加字段。
- `expiresAt` 用毫秒时间戳与 CLI 对齐。

## 6. 额度查询

### 6.1 接口

| 服务 | 请求 | 关键 Header |
|---|---|---|
| Codex | `GET https://chatgpt.com/backend-api/wham/usage` | `Authorization: Bearer …`、`ChatGPT-Account-Id`、`User-Agent: codex-cli` |
| Claude | `GET https://api.anthropic.com/api/oauth/usage` | `Authorization: Bearer …`、`anthropic-beta: oauth-2025-04-20`、`User-Agent: claude-code/2.1.0` |

两边响应都映射到统一的 `QuotaSnapshot { app, fiveHour, weekly, weeklyOpus?, weeklySonnet?, weeklyDesign?, planType?, fetchedAt }`。`QuotaWindow.usedPercent` 来自原始字段（Codex 是 `used_percent`，Claude 是 `utilization`），`remainingPercent = 100 − usedPercent`。Claude Design 周额度映射到 `weeklyDesign`，当前主字段为 `seven_day_omelette`，并保留若干旧猜测字段作为容错。

### 6.2 调用策略

- 启动只读 `quota-cache.json` 灌进 AppState，**不**立刻打网络。
- 周期循环按设置间隔触发（默认 5 分钟，可关闭）；任意一家上次成功 < 60s 时跳过。
- 收到 429 后该服务进入 10 分钟退避，期间所有触发短路。
- 手动刷新绕过 60s 节流，但仍遵守 429 退避。
- 401 / 403 不会自动停止循环（保持简单），但错误会显示在 Popover。
- 网络/解析失败保留旧快照，错误写入 `lastError`，下一 tick 重试。

### 6.3 Claude CLI 兜底

`ClaudeCLIFallbackQuotaClient` 仅在 `refreshNow` 触发 Claude API 失败 **且** `claudeQuota == nil` 时启动，且自身有 10 分钟独立冷却：

- 候选可执行：`/opt/homebrew/bin/claude` → `/usr/local/bin/claude` → `/usr/bin/claude` → `env claude`。
- 用 `/usr/bin/script -q /dev/null <claude>` 包一层伪 PTY 拿到 ANSI 输出；失败再裸跑一次直接执行。
- 通过 stdin 写入 `/usage\n\n/status\n/exit\n` 驱动 CLI。
- 输出 strip ANSI，按 `Current session` / `Current week (all models)` / `Current week (Sonnet only)` / `Current week (Opus)` 标签抓百分比与 `Resets …` 文本。
- 错误关键词识别：`rate_limit_error` / `authentication_error` / `failed to load usage data`，转成对应 `QuotaError`。

CLI 兜底拿到的 snapshot 用 `QuotaSnapshotSource.cliFallback` 标记，仅用于救急展示。

### 6.4 持久化

`QuotaCache` 写 `~/Library/Application Support/com.nanvon.ccbar/quota-cache.json`，结构 `{ codex: Record?, claude: Record? }`，每条记录含 snapshot、source、updatedAt。

## 7. 本地 JSONL 扫描

### 7.1 Claude（`ClaudeJSONLScanner`）

- 入口路径 `~/.claude/projects`，遍历所有 `.jsonl`。
- 每条 `type=assistant` 行解析 `message.id` / `message.model` / `message.usage`（`input_tokens` 不含 cache_read）/ `stop_reason`。
- 本批次内按 `message.id` 取唯一，优先选 `stop_reason != nil` 或更高 output_tokens 的一条；最后与全局 `seen` 集合比对，避免 sidechain / subagent 文件中重复计数。
- 全局 seen 列表持久化在 `scan-state.json`，上限 20000，超出截尾。

### 7.2 Codex（`CodexJSONLScanner`）

- 入口路径 `~/.codex/sessions` + `~/.codex/archived_sessions`。
- `turn_context` 行更新当前模型（写入 `ScanFileState.lastModel`）。
- `event_msg` 且 `payload.type == "token_count"` 行，从 `payload.info.last_token_usage` 直接累计（不是累计 delta），`info == nil` 跳过。
- `billable_input = input_tokens − cached_input_tokens`（Codex 的 `input_tokens` 含 cache_read）。

### 7.3 增量与持久化（`ScanCache`）

- 每个文件维护 `ScanFileState { mtime, offset, lastModel? }`。
- 调度逻辑：`mtime == prev && offset == size` → 跳过；`offset > size` → 文件被截断，归零重扫（去重靠 seen 集合）；否则从 `offset` 处续读到末尾。
- 已不存在的文件 watermark 在每次扫描末尾清理。
- 顶层结构 `ScanState { version, claude, codex, claudeSeenMessageIds }`，版本号 v3。`UsageRollupPayload { version, buckets, updatedAt }` 同步用 v3。版本不一致时整份丢弃以触发全量重扫。

### 7.4 聚合（`UsageAggregator`）

- `@Observable @MainActor`，内存 `[BucketKey: UsageBucket]`，BucketKey = (day, app, model)。
- `ingest(entries)` 累加 token 与 cost；`snapshot()` 返回数组用于落盘；`todayCost(for:)` / `totals(app:from:to:)` 给 UI 用。
- 由于是 `@Observable`，Popover 和统计 tab 在 body 读 `aggregator.totals(...)` 时自动建立依赖，新扫描完写回 buckets 后 UI 自动刷新。

### 7.5 价格（`Pricing`）

- 表硬编码于 `Pricing.swift`，命中不到的 model cost = 0，token 仍记录。
- 归一化：剥 `openai/` 前缀；Vertex 风格 `name@YYYYMMDD` 截断 `@` 前；Anthropic 风格 `-YYYY-MM-DD` 或 `-YYYYMMDD` 后缀剥掉；最后 `lowercased()`。
- 金额用 `Decimal`（`Foundation.Decimal`），单位 USD / 百万 token。

### 7.6 协调（`UsageService`）

- `bootstrap` 在 AppState 启动时读取 `usage-rollup.json` 灌进 aggregator，立即对外发布今日 cost。
- `scanNow()` 防重入：从 `ScanCache` 读旧 watermark，Claude / Codex 扫描在 detached `.utility` 优先级 Task 并行执行，`ingest` 后写 `scan-state.json` 与 `usage-rollup.json`，更新 `lastScanAt` 并 publish today cost。

## 8. 调度（`Scheduler`）

`@MainActor`，持有 AppState 弱引用，两条独立 `Task` 循环：

- **额度循环**：`quotaInterval` 秒触发一次 `appState.refreshQuotas(reason: .periodic)`。`nil` 或 0 时不启动（对应设置项「关闭」）。
- **用量循环**：`usageInterval` 秒触发一次 `appState.usageService.scanNow()`。

`setQuotaInterval` / `setUsageInterval` 在间隔变化时取消旧 task 并按新参数重启，对应设置变更的实时生效。

## 9. UI 层

### 9.1 菜单栏 label

`MenuBarLabel` 用 SwiftUI 包一层 `Image(nsImage:)`，真正画图在 `MenuBarBadgeImage.make`：

- 固定高 22pt；每段：18pt logo + 3pt gap + 文字（`NSFont.menuBarFont` + `monospacedDigit`）。
- 段间距 9pt。
- Logo 从 `Resources/Logos/codex.svg|claude.svg` 加载，缓存在 `NSCache`。
- 输出 `isTemplate = true`，系统在深色菜单栏自动取反。
- 用户都关闭时返回单 logo 占位，避免菜单栏空白。

### 9.2 Popover (`PopoverRootView`)

- 宽 340pt；header（标题 + 副标题 + 状态点 + 刷新/统计/设置/⋯）+ 两个 `ServiceBlockView`，中间 Divider。
- 状态点 (`headerState`)：取两边 `lastSuccessAt.max()`，按当前刷新间隔 ×1.5 / ×3 阈值判 live / stale / offline。
- `ServiceBlockView`：5h 大字百分比 + 5h 进度条 + reset 倒计时 + today/week cost；下一行 weekly 进度 + 百分比 + reset。Cost 显示 `<$1` / `$N` 整数。
- 颜色由 `Main/DesignSystem.swift` 的 `statusColor(remainingPercent:tint:)` 给出。

### 9.3 主窗口

- `MainWindowRootView` 是 TabView，`@Bindable` 绑到 `appState.mainTab`，所以 ⌘1 / ⌘, 切换是把 AppState 的字段改了。
- 统计 tab：见 [产品需求.md](产品需求.md) §7；数据全部来自 `UsageAggregator.totals(...)`，自定义区间用本地 startOfDay；上一同长度区间用于 KPI delta。
- 设置 tab：自定义 `PrefsGroup` / `PrefsRow` 卡片样式（系统 `Form .grouped` 在这套设计下太弱），所有 Toggle / Picker 直接绑 `SettingsStore`；间隔变化时调用 `appState.applySettingsChange()`。

### 9.4 悬浮窗

- `FloatingPanel`（`NSPanel` 子类）：`.borderless` + `.nonactivatingPanel` + `level = .floating` + `isMovableByWindowBackground = true`。
- `FloatingPanelController`：维护一个 NSPanel + NSHostingView<FloatingContentView>；用 SwiftUI `fittingSize` 双帧同步把窗口缩到内容大小。
- 拖动结束（用 200ms debounce 模拟）后判断离屏幕边缘距离 < 20pt 触发吸附。
- frame 写回 `SettingsStore.floatingPanelFrame`，启动时若 frame 不在任何屏幕上回落到默认右上角。

### 9.5 引导

- 4 步 (`WelcomeStep` / `DetectAccountsStep` / `ConfigureStep` / `ReadyStep`)，进度由本地 `@State step: Int` 控制。
- `ReadyStep` 完成后写 `didCompleteOnboarding = true` 并关闭窗口；可直接跳转到主窗口的统计 tab。

### 9.6 设计系统（`Main/DesignSystem.swift`）

公共组件集中在此：

- `statusColor(remainingPercent:tint:)`：≤0 红 / <20 橙 / ≥20 服务色。
- `Color.ccPanel(cornerRadius:)` / `ccPanelStroke`：浅色白底 / 深色 23.5% 灰 + 6% opacity 描边。
- `ServiceMark`（8×8 squircle）、`ServiceTile`（带 logo 的圆角图块）。
- `ProgressBar`：服务色填充 + 5%/8% 浅深底色。
- `VisualEffectBackground`：包 `NSVisualEffectView` 给悬浮窗 / 引导使用。

## 10. 国际化（`Core/L10n.swift`）

- `tr(en, zh)` 全局函数，`@MainActor`，按 `SettingsStore.resolvedLanguage` 选词。

- `formatResetHint` / `formatResetCompact` 把重置时间格式化为 `5h 7m 后重置` / `5h7m` 或绝对时间，受 `resetTimeDisplay` 设置控制。

## 12. 并发与线程

- AppState / UsageAggregator / UsageService / Scheduler / FloatingPanelController 全部 `@MainActor`。
- 凭据读取、token 续期、网络请求、JSONL 扫描全部走 `Task.detached(priority: .utility)`，结果回到 MainActor 写入状态。
- `Quota` 系列 client 全部 `nonisolated static`，不依赖 actor 隔离。
- 不使用 GCD；不使用第三方并发库。

## 13. 已知技术债 / 持续关注

- **Claude CLI 兜底依赖外部进程**：用户没装 CLI 时会失败，目前只在 API 失败且无缓存的手动刷新触发，影响面有限。
- **价格表手维护**：模型更新需要同步 `Pricing.swift`；版本号 v3 用于在表对齐时丢弃旧桶并重扫。
- **seen ID 上限 20000**：极端长时间使用 + 大量 sidechain 引用可能导致环形丢失；目前足以覆盖正常使用。
- **没有自动更新通道**：发布要手动出包；后续可接 Sparkle。
