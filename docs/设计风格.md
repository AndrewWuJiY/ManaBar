## 0. 路线:对齐 prototype,系统默认作为底座

cc-bar 是高频菜单栏入口,Popover 5 秒钟的扫视体验决定了产品观感。系统默认 + 数据列表只能做到"够用",做不到"一眼看到最关心的"。所以所有界面均对齐设计稿 [Prototype.html](历史参考/设计稿/Prototype.html) 落地,关键信息层做自绘组件,非关键交互一律用系统默认。

落地原则:

1. **能用系统的就用系统的**(material / Form / Picker / Switch / DatePicker / GroupBox 都不重绘),作为视觉底座。
2. **prototype 里承担信息层级的自绘组件全部做**:进度环、weekly 条、服务 tile、KPI 大字 + delta chip、堆叠柱图、双行 pill HUD。这些是"焦点 / 比例 / 趋势"的视觉载体,不是装饰。
3. **prototype 里纯装饰的细节按取舍表处理**(下表)。

| 自定义项 | 落地 | 备注 |
|---|---|---|
| 产品识别色(Codex 石墨灰 / Claude 桃橙) | ✅ | 见 §4.2 |
| 双语版式(英文主 + 中文次) | ✅ | 见 §5 |
| 状态色(4 档:>50% normal / 20~50% warning / <20% low / =0 empty) | ✅ | 见 §4.3 |
| Tabular 数字 | ✅ | 见 §3 |
| 进度环 / Weekly 条 / 服务 tile / KPI 大字 + delta chip | ✅ | 见 §11 |
| 堆叠柱图 / 甜甜圈 / 面积线 / 热力图 | ✅ | 用 Swift Charts;热力图可自绘 |
| Sidebar + Panel 卡片(Stats / Settings) | ✅ | 见 04 |
| Toggle / Dropdown / 自定义 Slider | ✅ | 见 04 §4 |
| SF Symbols 图标 | ✅ | 见 §9 |
| 顶部 1pt 高光线(模拟玻璃反光) | ❌ | 系统材质已有玻璃感,边际收益低 |
| 内描边(`inset 0 0 0 0.5`) | ⚠️ | 大窗口/HUD 不做;Panel / KPI 卡需要描出"卡片感"时叠 0.5pt(见 §7) |
| 三层精确阴影 | ⚠️ | 系统优先;HUD 不够再叠 fallback(见 §7) |
| 自定义动画曲线 `cubic-bezier(0.2, 0.9, 0.3, 1.15)` | ⚠️ | Popover 出现可用,其余系统默认(见 §8) |
| 8 种菜单栏 / 8 种 HUD 变体 | ⚠️ | 默认各 1 种推荐变体;其余作为 Settings 选项(见 04 §4) |
| App 图标正式 `.icns`(渐变石墨→桃) | ✅ | 见 §9 + Onboarding |

## 1. 整体气质

- **关键词**:克制、原生、半透明磨砂、连续曲率(squircle)、**强信息层级**(焦点突出、次级弱化)。
- **禁用**:Material Design / Bootstrap / Fluent 风格的自造按钮、彩色阴影、霓虹高光、emoji 状态指示。
- **遵循**:macOS 26 Tahoe HIG。所有交互控件(Switch / Picker / DatePicker / SegmentedControl)用系统原生外观,不重绘。

---

## 2. 材质

全部用系统给的,不自己叠玻璃效果。

| 场景 | material |
|---|---|
| 菜单栏弹出面板 | `.popover` |
| 桌面悬浮窗 HUD | `.hudWindow`(`NSVisualEffectView`,blendingMode `.behindWindow`) |
| 主窗口侧边栏 | `.sidebar` |
| 主窗口卡片 / 分组 | SwiftUI `.regularMaterial` |

SwiftUI 侧:`.background(.regularMaterial)` / `.thinMaterial`。
AppKit 侧:`NSVisualEffectView`,`blendingMode = .behindWindow`。

**HUD 实现注意**:`NSPanel` 默认 background 是 `.clear`,需要把 `NSVisualEffectView(.hudWindow)` 作为 panel content 的最底层,而**不是**在 SwiftUI 里手画 `RoundedRectangle.fill(.regularMaterial)`。

---

## 3. 字体

- 全部 `-apple-system` / `.systemFont`,不指定字族(系统自动切 SF Pro Display / Text)。
- **所有数字** tabular:SwiftUI `.monospacedDigit()`,AppKit `NSFont.monospacedDigitSystemFont(...)`。

### 字号体系(基于 prototype)

| 角色 | 字号 / 字重 / 字间距 | 用途 |
|---|---|---|
| KPI 大数字 | 28pt / .semibold / kerning -0.5 / 默认 design | Stats hero、Popover big stat |
| Title | 17pt / .bold / kerning -0.4 | 窗口标题(如 `Statistics · 用量统计`) |
| Headline | 13pt / .semibold / kerning -0.1 | 节标题、Popover 服务名 |
| Body | 13pt / .regular | 正文标签 |
| Callout | 12.5pt / .regular | 紧凑正文 |
| Caption | 11pt / .regular / secondary | 副标、说明 |
| Mono small | 10pt / .regular / .monospaced design | 时间戳、路径 |
| Eyebrow | 10.5pt / .semibold / kerning +0.4 / 大写 | 小标签(`CODEX` 等) |

> SwiftUI 实现:`.kerning(...)` 等价于 prototype 里的 `tracking`,SwiftUI 没有 tracking API。
> 默认行高交给系统,不指定 `lineSpacing`。

## 4. 颜色

### 4.1 基础语义色

| 用途 | Token |
|---|---|
| 主文本 | `.labelColor` / SwiftUI `.primary` |
| 次要文本 | `.secondaryLabelColor` / `.secondary` |
| 三级文本(中文副标 / caption 二级) | `.tertiaryLabelColor` |
| 分隔线 | `.separatorColor` |
| 浅色填充 | `.quaternarySystemFill` / `.secondarySystemFill` |
| Accent | `systemBlue`(系统自动) |

### 4.2 产品识别色(唯一两个写死的颜色)

| 服务 | 浅色 | 深色 |
|---|---|---|
| Codex | `#6C6C70`(石墨灰) | `#98989D` |
| Claude Code | `#D97757`(桃橙) | `#E68A6E` |

**用在哪**:服务色点 / 服务 tile / logo / 堆叠柱图块 / 图例等服务识别元素。额度数字、进度环、weekly 条、HUD 百分比等额度状态载体统一走 §4.3 状态色,**不要**再用服务识别色。**不要**用在大面积窗口背景。

**渲染顺序**:Codex 永远第一,Claude Code 第二。

> 实现:在 `Resources/Assets.xcassets` 里定义了 `CodexAccent` / `ClaudeAccent` 两个 Color Set(浅 / 深各一套),Xcode 自动生成 `Color.codexAccent` / `Color.claudeAccent`,直接使用即可。

### 4.3 状态色

额度状态色按剩余比例分 **4 档状态色**,全局唯一来源是 `statusColor(remainingPercent:tint:)`(`Main/DesignSystem.swift`),Popover / Floating / Stats KPI 全部走它。状态色保留 normal / warning / low / empty 语义,但避免使用系统原色造成警报感。产品识别色(Codex 石墨灰 / Claude 桃橙)只用于服务标识(tile / logo / 图表),**不再**参与额度状态着色;`statusColor` 的 `tint` 参数保留但当前不使用,以备将来切回「服务色打底」方案。

| 状态 | 颜色 |
|---|---|
| 剩余 > 50% | normal:浅色 `#64748B` / 深色 `#A7B3C2` |
| 剩余 20% ~ 50% | warning:浅色 `#F6C343` / 深色 `#FFE27A` |
| 剩余 < 20% | low:浅色 `#FF7A2F` / 深色 `#FFA15F` |
| 剩余 = 0 / 超限 | empty:浅色 `#FF4D6D` / 深色 `#FF7A90` |
| live(刷新正常) | `systemGreen` 小点 |
| stale(数据过期) | `systemOrange` 小点 |
| offline(连续失败) | `systemRed` 小点 |
| 服务状态 none(statuspage 正常) | `systemGreen` 6pt 圆 |
| 服务状态 minor | `systemYellow` 6pt 圆 |
| 服务状态 major | `systemOrange` 6pt 圆 |
| 服务状态 critical | `systemRed` 6pt 圆 |
| 服务状态 maintenance | `systemBlue` 6pt 圆 |
| 服务状态 unknown / 未拉到 | 不渲染 |

**菜单栏例外**:菜单栏图标使用 `isTemplate = true` 模式,只能黑/白随菜单栏深浅自动反色。**状态色规则不作用于菜单栏文字本身**;低额度提示通过 Popover header 状态点、悬浮窗百分比颜色、未来的通知体现。

## 5. 语言切换

### 5.1 通用版式

- **单语显示**:同一时间只渲染中文或英文,不再并列双语。
- 由 `Settings → General → Language` 控制,三档:`Follow System` / `中文` / `English`。默认 `Follow System`,根据 `Locale.preferredLanguages.first` 是否以 `zh` 开头解析为 zh / en(兼容简繁)。
- 设计文案仍按"主英 + 中文"成对编写,运行时由 `tr(en, zh)` 选词;不直译,中文要短而地道。
- **所有可见文案**都要写双语对照(en + zh),用于翻译;不允许只有单一语言的硬编码。

### 5.2 典型组合

| 场景 | 形态 |
|---|---|
| 节标题 | `tr("Menu Bar", "菜单栏")` |
| 行内标签 + 副标 | `Codex` 主行 + 下一行 `tr("OpenAI", "OpenAI · Plus")` |
| 描述段落 | 一段,按 language 选英文段或中文段 |
| 按钮 | `tr("Open Statistics", "查看统计")` |
| 空状态 | `tr("Waiting for data", "等待数据")` |

### 5.3 实现

- `Core/L10n.swift` 提供 `AppLanguage` enum 与全局 `tr(_:_:)` 函数。
- 调用方在 SwiftUI view body / 函数体内统一写:
  ```swift
  Text(tr("Menu Bar", "菜单栏"))
  ```
- `SettingsStore.shared.appLanguage` 持久化用户选择;`resolvedLanguage` 计算属性把 `.system` 解析为 `.zh / .en`。
- `BilingualInline` / `BilingualStack` 已改造为按 language 单语渲染,外部 API 兼容。
- AppKit `NSMenu`(Commands API、NSAlert)在切换 language 后,通常需要重启应用才能完全刷新;SwiftUI 视图(Popover、HUD、主窗口、Settings、Onboarding)运行时即时刷新。

## 6. 圆角

绝大多数用系统默认(`NSPopover` / `NSPanel` / `Window` 自己就有正确的连续曲率圆角)。需要自绘时按下表:

| 元素 | 半径 |
|---|---|
| Window | 12pt(系统给) |
| Popover | 18pt(系统给) |
| HUD | 14pt(`NSPanel` mask 或最底层 NSVisualEffectView 的 `maskImage`) |
| 卡片 / 分组 | 10–12pt |
| 服务 tile(22pt 方块) | 6pt |
| 行内 pill / 小标签 | 6–8pt |
| Weekly 进度条 | 2.5pt(条高 5pt 的一半) |

实现统一用 `RoundedRectangle(cornerRadius: ..., style: .continuous)`。

---

## 7. 阴影

**优先靠系统**。`NSPopover` / `NSPanel(.floating)` / `Window` 自带阴影。

**HUD 例外**:`NSPanel` + clear background 时系统阴影可能不足,在最底层 `NSVisualEffectView` 容器或最外层 SwiftUI view 上叠一层:

```swift
.shadow(color: .black.opacity(0.28), radius: 14, y: 10)
```

不再叠"内描边模拟高光"。

---

## 8. 动画

默认用 SwiftUI / AppKit 系统动画。下表是允许的自定义:

| 场景 | 时长 + 曲线 |
|---|---|
| 刷新图标旋转 | 700ms ease-in-out,转 1 圈 |
| Popover 出现(可选) | 180ms `timingCurve(0.2, 0.9, 0.3, 1.15)`,`scale(0.96) → 1.0`,带 6pt y 偏移 |
| Toggle / hover / 切换 | 系统默认,不调参(`Toggle` 颜色覆盖见 §11.9) |

> Popover 出现动画属于"锦上添花"。如果用 `MenuBarExtra(.window)`,系统已有自己的过渡;只有用自绘 `NSPanel` 时才需要写。

---

## 9. 图标

- **一律 SF Symbols**,不自造 SVG(服务 logo 例外:Codex / Claude 用自带 SVG,见 `Resources/Assets.xcassets`)。
- 常用清单:`gauge.medium` / `arrow.clockwise` / `chart.bar.xaxis` / `gearshape` / `xmark.circle.fill` / `line.3.horizontal` / `ellipsis.circle` / `circle.fill`(状态点) / `square.and.arrow.up` / `power`。
- **菜单栏图标用 template 模式**(`isTemplate = true`),自动适配深浅菜单栏。
- **App 图标**:已经出正式版,1024 / 512 / 256 / 128 / 64 / 32 / 16 多档存放在 `Resources/Assets.xcassets/AppIcon.appiconset/`(渐变石墨灰 → 桃橙),Xcode 自动按需取用。

## 10. 间距

4pt 基线:**`{2, 4, 6, 8, 10, 12, 14, 16, 20, 24, 28, 32}`**。

- 卡片内 padding:14pt(Popover 服务 block)/ 16pt(主窗口卡片)。
- 卡片间间距:10pt(Popover 服务之间)/ 12pt(主窗口卡片之间)。
- 节标题与正文间距:10–12pt。
- 行内元素间距:6 / 8pt。
- 图标与文字间距:6 / 8pt。

**禁止**出现表外奇数尺寸(7pt / 9pt / 11pt 等)作为间距。
**例外**:具体控件的固定宽度(如百分比右对齐 frame `width: 42`、5h/1w 标签 frame `width: 24`)不在此约束内。

## 11. 组件尺寸约定

prototype 里的关键自绘组件,固化尺寸:

### 11.1 服务标识(色点 / 色方)

prototype 里"色点"的实际形态分两种,**不要混用**:

| 用途 | 形态 | 尺寸 |
|---|---|---|
| Stats sidebar Service 行 / By-service 列表 / KPI 卡 dot / 图例 | **8×8 squircle,圆角 2pt**(轻微圆角的方块,**不是圆**) | 8pt |
| Daily trend 行内 / Pie 图例 | 同上 | 6–8pt |
| KPI 卡 label 前缀色块 | 同上,更小 | 6×6 圆角 1.5pt |
| Popover header 状态点(live/stale/offline) | **完整圆** | 7pt 直径 |
| Onboarding "Connected" 状态点 | **完整圆** | 6pt 直径 |

> 实现:服务色块用 `RoundedRectangle(cornerRadius: 2, style: .continuous)`;状态色点用 `Circle()`。
> SwiftUI 别图省事统一用 Circle,会和 prototype 不一致。

### 11.2 服务 tile(带 logo)

- 尺寸:22pt × 22pt
- 圆角:6pt continuous
- 填充:
  - Codex:还原 OpenAI 官方品牌图标——纯白底 + 黑色 logo + `Color.black.opacity(0.12)` 0.5pt hairline 边框(避免白底融进浅色背景)
  - Claude:服务识别色(桃橙)@ 100%,白色 logo
- 内 logo:14pt 居中
- 出现位置:Popover 服务行左侧、Stats sidebar、Onboarding 账号列表
- 注:tile 是唯一对 Codex 用白底的位置;图表、图例等服务识别元素里的 `Color.codexAccent` 仍是石墨灰,额度数字和额度进度条不再用它着色

### 11.3 进度环

按场景分三档:

| 场景 | 直径 | 描边 | 中央内容 |
|---|---|---|---|
| Popover 服务 5h | 56pt | 5.5pt | 百分比 14pt .semibold + `5H` 8pt tertiary letterSpacing 0.2 |
| Stats Current limits 4 个小环 | 32pt | 4pt | 整数(无 %)9.5pt .semibold |
| HUD variant 3 dual mini rings | 26pt | 3.5pt | 无(右侧另起百分比) |
| HUD variant 5 single service | 34pt | 4pt | 整数 10pt .semibold |

- 颜色:状态色(4 档 normal / warning / low / empty,见 §4.3)
- 背景轨道:`.quaternarySystemFill`,同宽
- 实现:SwiftUI `Canvas` 或 `Circle().trim(from:to:).stroke(style: StrokeStyle(lineWidth:..., lineCap: .round))`

### 11.4 进度条 Bar

按场景分多档(prototype 用得很细):

| 场景 | 高 | 圆角 |
|---|---|---|
| Popover 5h(weekly 条) | 5pt | 2.5pt |
| Stats by-service 主条 | 5pt | 2.5pt |
| Stats KPI / Popover 周额度条 | 5pt | 2.5pt |
| HUD bar | 4pt | 2pt |
| Popover dense compact 卡内主 bar | 4pt | 2pt |
| Popover dense compact 周 bar | 3pt | 1.5pt |
| Popover BigStat 主 bar | 6pt | 3pt |

- 颜色:状态色;轨道 `.quaternarySystemFill`
- 标签:`Weekly · 周额度` 左对齐 caption + 百分比右对齐 caption .semibold tabular

### 11.5 KPI 大字 + Delta

按场景分两档:

| 场景 | 数字字号 | Delta |
|---|---|---|
| Stats 顶部 4 个 KPI 卡 | **22pt** .semibold kerning -0.5 tabular | 11pt .semibold,`↑ 12.4%` 紧贴数字右,不用 Capsule 背景。red / green 文字直接铺 |
| Popover BigStat 大数字(variant D) | 38pt .semibold kerning -1 | — |
| Popover SparkChart 头部金额 | 22pt .semibold kerning -0.4 | 12pt .semibold green 紧贴右 |

注意:**Delta 不是 chip / Capsule**,prototype 是裸文字 `↑ 12.4%`,颜色按中国习惯红涨绿跌。SwiftUI 实现别给它加背景。

### 11.6 堆叠柱图(Stats)

- 高:160pt
- 每根条:`flex 1`(平均分宽),3pt 间隙
- **栈顺序**:Claude 在**上**(`D97757` / `E68A6E`)、Codex 在**下**(浅色 `6C6C70` / 深色 `98989D`),栈内 1pt gap 分隔色块
- 圆角:栈顶 segment 顶 2pt(`2px 2px 0 0`);栈底 segment 底 2pt(`0 0 2px 2px`);**栈内不全圆角**
- `min-height: 1pt`(确保零值也可见为细线)
- 实现:Swift Charts `BarMark` + `.foregroundStyle(by:)` + `clipShape`(顶/底 segment 自定义圆角)

### 11.7 Popover

- 宽:340pt
- 高:auto(双服务 ~360pt;单服务 ~210pt)
- 圆角:18pt(系统 `NSPopover` 提供)
- 内 padding:见 §10 / `04-界面布局.md`

### 11.8 悬浮窗 HUD

- 最小宽:168pt
- 圆角:14pt
- 内 padding:10pt 上下 / 14pt 左右
- 行间距:7pt(本档允许,作为 HUD 紧凑特例)

### 11.9 Toggle 与 hover / 光标

- **Toggle 颜色**:统一 `.toggleStyle(.switch).tint(.green)`,打开态走系统 `Color.green`(浅 `#30A14E` / 深 `#30D158`,见 §12.1)。Codex / Claude 账户开关也用绿色,**不**用产品识别色。这是「打开 = 启用 / 安全」的语义,产品识别色仅用于服务标识与按服务图表等数据可视化场景。
- **hover 手型光标**:所有 `.borderless` / `.plain` 自定义按钮(Popover 顶部图标、Stats sidebar 条目、Onboarding 主次按钮、Settings 内 Account 行等)统一加 `pointingHandCursor()` ViewModifier(实现见 `Main/DesignSystem.swift`),鼠标进入时切换 `NSCursor.pointingHand`,离开时还原。系统原生 `.bordered` / `.borderedProminent` / `Toggle` / `Picker` **不**额外加,沿用系统行为。
- **hover 背景**:仅 `PopoverIconButtonStyle`(Popover 顶部图标)采用浅灰圆角背景(`Color.primary.opacity(0.08)`,圆角 5pt)。其它按钮按各自的 hover 风格自治,不强制。

## 12. 浅色 / 深色对照

prototype 提供了 light + dark 两套效果(Canvas.html 左右对照)。绝大多数差异 SwiftUI 用系统材质 / 语义色就自动适配,**只需特别注意以下几点**。

### 12.1 自动适配的(不用手写)

| 项 | 浅色 | 深色 | SwiftUI 怎么写 |
|---|---|---|---|
| 主文本 | `#1D1D1F` | `#F5F5F7` | `.primary` |
| 次要文本 | `rgba(60,60,67,0.6)` | `rgba(235,235,245,0.6)` | `.secondary` |
| 三级文本 | `rgba(60,60,67,0.3)` | `rgba(235,235,245,0.3)` | `.tertiary` / `Color.primary.opacity(0.3)` |
| 分隔线 | `rgba(60,60,67,0.12)` | `rgba(255,255,255,0.1)` | `Divider()` / `.separatorColor` |
| 强分隔线(checkbox 描边等) | `rgba(60,60,67,0.29)` | `rgba(255,255,255,0.16)` | `.tertiary` 或 `Color.primary.opacity(0.16)` |
| 浅色填充 fill | `rgba(120,120,128,0.16/0.08/0.04)` | `rgba(120,120,128,0.24/0.16/0.08)` | `.quaternarySystemFill` 三档 |
| Accent | `#007AFF` | `#0A84FF` | `.accentColor` / `Color.accentColor` |
| System Green / Yellow / Orange / Red | 系统自适配 | 系统自适配 | `Color.green` / `Color.yellow` / `.orange` / `.red` |
| Quota normal / warning / low / empty | `#64748B` / `#F6C343` / `#FF7A2F` / `#FF4D6D` | `#A7B3C2` / `#FFE27A` / `#FFA15F` / `#FF7A90` | `statusColor(remainingPercent:tint:)` |
| 窗口 / Popover / HUD 背景 | 浅色 frosted | 深色 frosted | `.background(.regularMaterial)` / `NSVisualEffectView` |

### 12.2 产品识别色(双套已定义,自动切换)

| 服务 | 浅色 | 深色 |
|---|---|---|
| Codex | `#6C6C70` | `#98989D` |
| Claude Code | `#D97757` | `#E68A6E` |

实现见 `Resources/Assets.xcassets/CodexAccent.colorset` / `ClaudeAccent.colorset`,通过 `Color.codexAccent` / `Color.claudeAccent` 调用。**Popover 服务 tile / logo / 图表服务块 / 图例** 用这两个色,系统会按当前外观切换；额度数字、进度环、weekly 条、HUD 百分比统一用 `statusColor`。

### 12.3 需要手写的差异(浅深色不一致,系统语义色覆盖不到的)

**Panel / 卡片背景**(Stats / Settings / Onboarding 都用):

| 模式 | 背景 |
|---|---|
| 浅色 | 实心 `#FFFFFF`(不透明) |
| 深色 | `rgba(60,60,63,0.4)` 半透 |

SwiftUI 实现:
```swift
@Environment(\.colorScheme) private var colorScheme

var panelBg: AnyShapeStyle {
    colorScheme == .dark
        ? AnyShapeStyle(Color(white: 0.24, opacity: 0.4))
        : AnyShapeStyle(Color.white)
}
```
再加 0.5pt 内描边:
```swift
.overlay(
    RoundedRectangle(cornerRadius: 10, style: .continuous)
        .strokeBorder(
            colorScheme == .dark
                ? Color.white.opacity(0.06)
                : Color.black.opacity(0.06),
            lineWidth: 0.5
        )
)
```

> 这是 prototype 给"卡片感"的唯一手段(没有阴影)。不加描边深色卡片会"飘"在窗口里。

**HUD 背景**:

| 模式 | 背景 alpha |
|---|---|
| 浅色 | `rgba(255,255,255, 0.58)` + 40px blur + 180% saturate |
| 深色 | `rgba(28,28,30, 0.62)` + 同上 |

实现:用 `NSVisualEffectView(material: .hudWindow, blendingMode: .behindWindow)`。`.hudWindow` material 已经自动满足上述两套 alpha + blur,**不要在 SwiftUI 里再叠 `.background(.regularMaterial)`**。

**Segmented control**(Stats toolbar 时间范围):

| 模式 | 容器 bg | active segment bg |
|---|---|---|
| 浅色 | `rgba(0,0,0,0.06)` | `#FFFFFF` + 微阴影 `0 0.5 2 rgba(0,0,0,0.18)` |
| 深色 | `rgba(255,255,255,0.08)` | `rgba(120,120,128,0.55)` + 同微阴影 |

SwiftUI 推荐**直接用系统 `Picker(.segmented)`**,会自动适配,不要自绘。如果一定要自绘(需要 7 个选项 + 移动指示),按上表写。

**Sparkbar / Stats stacked bars 颜色**:无论浅深都用 prototype 的硬编码值:

| 服务 | 浅色 stack | 深色 stack |
|---|---|---|
| Codex | `#6C6C70` | `#98989D` |
| Claude | `#D97757` | `#D97757`(深色不变,因为图表自身较暗时桃橙仍清晰) |

> 注意:**图表用桃橙在深色不切换到 `#E68A6E`** —— prototype 在堆叠柱图里 Claude 段强制用浅色模式那个值,以保证视觉饱和度。Codex 段反之,按外观切换。这是个例外,实现时不要图省事统一切色。

### 12.4 状态点 / 标签可读性

- HUD 在深色玻璃上,状态点直接放系统色即可;不需要额外描边。
- 浅色模式下,状态点周围的 `Connected · 已连接` 文字用 systemGreen,**不**用 .primary,以保持和点的一致性。
- KPI 卡的 delta `↑ 12.4%` 浅色用系统 red,深色也随系统 red 自动切换;`↓ 2.3%` 用系统 green。**不**用产品色。
