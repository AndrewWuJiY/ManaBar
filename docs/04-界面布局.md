# cc-bar 界面布局草案（粗略版）

> 对应 [01-需求.md](01-需求.md) / [02-实现.md](02-实现.md) / [03-设计风格.md](03-设计风格.md)。
> 本文档只列**每个界面有哪些区块、信息层级、入口关系**。不写具体尺寸、不写控件 API、不画像素稿。

---

## 0. 界面总览

cc-bar 一共有 4 个可见界面：

| # | 名称 | 形态 | 入口 |
|---|---|---|---|
| 1 | 菜单栏图标 | `MenuBarExtra` / `NSStatusItem` | 常驻菜单栏 |
| 2 | 总览 Popover | 菜单栏弹出小面板 | 左键点击菜单栏图标 |
| 3 | 主窗口 | 普通窗口（含 2 个 tab） | Popover 底部「设置」按钮 |
| 4 | 桌面悬浮窗 | 无标题栏浮窗 | 设置里勾选开启 |

入口关系：

```
菜单栏图标 ──左键──> 总览 Popover ──设置按钮──> 主窗口 (统计 tab / 设置 tab)
                                  ──退出按钮──> 退出 App
                                  ──刷新按钮──> 触发一次数据刷新
                            
设置里勾选 ──> 桌面悬浮窗 显示/隐藏
```

---

## 1. 菜单栏图标

### 内容

- **图标**：SF Symbol 模板图标（深浅色菜单栏自动适配）。
- **图标右侧文字**（可选，由设置决定）：
  - 显示哪个应用：Codex / Claude / 两者
  - 显示哪种额度：5h 剩余% / 1w 剩余% / 两者
  - 都不勾 → 只显示图标
- 同时显示两个应用时，按固定顺序拼接：先 Codex 后 Claude（缩写形式待定，如 `C:34% · K:60%`）。

### 交互

- 左键 → 弹出总览 Popover
- 右键 / Ctrl+点击 → 暂不做（一期所有功能都从 Popover 进）

---

## 2. 总览 Popover

点击菜单栏图标弹出。系统默认 Popover 外观（Liquid Glass 材质）。

### 信息结构

```
┌─────────────────────────────┐
│ Codex 区块                   │
│   5h:   ▓▓▓▓░░░░  45%        │
│         reset in 2h 13m      │
│   1w:   ▓▓▓░░░░░  32%        │
│         reset in 4d 6h       │
│   今日 cost: $1.23           │
├─────────────────────────────┤
│ Claude 区块                  │
│   5h:   ▓▓▓▓▓▓░░  72%        │
│         reset in 1h 45m      │
│   1w:   ▓▓▓▓░░░░  48%        │
│         reset in 5d 2h       │
│   今日 cost: $2.41           │
├─────────────────────────────┤
│         [刷新] [设置] [退出]  │
└─────────────────────────────┘
```

### 每个应用区块展示

固定顺序：**先 Codex，再 Claude**。每块包含：

- **5h 窗口**：已用百分比 + 重置倒计时
- **1w 窗口**：已用百分比 + 重置倒计时
- **今日 cost**：当日折算花费（USD，本地 JSONL 解析）

> 两个应用各算各的，**不合计**。

### 底部按钮

从左到右：**刷新 · 设置 · 退出**。

- 刷新：触发一次额度 API + JSONL 增量扫描，按钮上短暂出现进度态
- 设置：打开主窗口，默认落在「统计」tab（见第 3 节）
- 退出：退出 App

### 不在 Popover 里展示的（已砍）

- 账号信息（邮箱 / plan）→ 挪到设置 tab 里展示
- 月度额外消费 / 本周本月 token 明细 → 全部进主窗口的统计 tab
- 「详细」单独入口 → 没有，统一从「设置」按钮进主窗口再切 tab

## 3. 主窗口

普通 macOS 窗口（系统默认窗口样式）。顶部 `TabView` 两个 tab：**统计 / 设置**。

**默认停在「统计」tab。**

> **实现说明**：用 SwiftUI 的 `Window` scene，**不要**用标准 `Settings { }` scene。后者是 macOS 系统约定的"⌘, 打开的偏好设置窗口"，只放设置；我们要统计和设置在同一窗口 tab 切换，所以走自定义 `Window`。

### 3.1 统计 tab

数据来源：纯本地 JSONL 解析。

区块结构（自上而下）：

1. **时间范围选择器**
   - 预设按钮：今天 / 本周 / 本月 / 最近 7 天 / 最近 30 天 / 全部
   - 自定义：起止日期两个 DatePicker
2. **汇总卡片**（Codex 和 Claude 各一张，**分开统计**）
   - 总 input / output / cache_read / cache_create token
   - 总花费 USD
3. **按模型拆分**（每个应用内部按 gpt-5 / claude-opus-4 / claude-sonnet-4 … 列出）
4. **按日趋势**（一期可先用列表，二期再上 Swift Charts 折线/柱状图）

### 3.2 设置 tab

5 个分组（参考 [01-需求.md](01-需求.md) 第 7 节）。本文档只列分组名，具体每项控件后续细化：

| 分组 | 大致包含 |
|---|---|
| **账号** | 显示已识别的 Codex / Claude 账号信息（邮箱 / plan）；勾选「显示哪些应用」 |
| **菜单栏** | 选择菜单栏图标右侧显示什么（哪个应用 / 哪种额度） |
| **悬浮窗** | 是否开启；显示哪些行（Codex 行 / Claude 行 / 两者） |
| **刷新** | 额度 API 刷新间隔（30s / 1min / 5min / 10min / 关闭）；JSONL 扫描间隔 |
| **通用** | 开机自启；关于（版本号、致谢、隐私说明） |

> 具体每个开关、下拉项的形态等做 UI 时再定，本文档不展开。

## 4. 桌面悬浮窗

可选开关，默认关闭。系统 `NSPanel` + `.hudWindow` material 或 `.regularMaterial`。

### 内容

- 默认两行：
  - 第 1 行：Codex 5h 剩余 %
  - 第 2 行：Claude 5h 剩余 %
- 可在设置里关掉其中一行 → 只显示一行。

### 形态

- 无标题栏、半透明圆角、不抢焦点
- 可拖动；位置记忆（下次启动还在原位）
- 置顶（默认开，后续可加开关）

---

## 5. 信息密度 / 状态色（跨界面统一）

按 [03-设计风格.md](03-设计风格.md) 锁的规则，跨所有界面统一：

- 数字一律 `.monospacedDigit()`，刷新不跳
- 百分比 < 20% → 橙色
- 百分比 = 0 / 已耗尽 → 红色
- 其余 → 默认前景色
- Codex 用石墨灰识别色，Claude 用桃橙识别色
