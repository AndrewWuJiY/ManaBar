# cc-bar 设计风格草案(粗略版)

> 对应 `01-需求.md` / `02-实现.md`。本文档只锁**设计风格**(气质 / 材质 / 字体 / 颜色 / 圆角 / 阴影 / 动画 / 双语规则),**不**写布局、不写每个界面长什么样(留给后续 `04-界面布局.md`)。
> 主要来源:`docs/参考/design_handoff_cc_bar/`(README.md + claude-code-prompt.md)。

---

## 0. 路线:系统默认 + 必要自定义

### 前提

**Liquid Glass 是 macOS 26 Tahoe 的系统默认设计语言**。只要用:

- `MenuBarExtra` / `NSPopover` 的默认外观
- `.background(.regularMaterial)`、`NSVisualEffectView` 的 `.popover` / `.hudWindow` / `.sidebar` material
- 系统的 `Form` / `GroupBox` / `Window` / `Settings` scene

**得到的就是 Liquid Glass**。这部分是"白送",不算自定义。`01-需求.md` 写的"系统默认"和参考文档"Liquid Glass"在这层不冲突。

### 一期取舍

参考文档在系统默认之上还要求了一堆"自定义层"。一期**只加必要的**,其余砍掉或推到二期。

| 自定义项 | 一期 | 理由 |
|---|---|---|
| 两个产品识别色(Codex 石墨灰 / Claude 桃橙) | ✅ 加 | 区分两个服务的核心识别,不加就只能靠文字 |
| 双语版式(英文主 + 中文次) | ✅ 加 | 需求里明确双语 |
| 状态色规则(<20% 橙、=0 红) | ✅ 加 | 用户最关心的"快没额度"信号 |
| 数字 tabular figures(`.monospacedDigit()`) | ✅ 加 | 不加刷新会跳,代价为 0 |
| 自绘进度环 / 进度条 / 堆叠条形图 / 甜甜圈 | ✅ 加 | 没现成控件,只能自绘;但用 Swift Charts 不是从零画 |
| SF Symbols 图标 | ✅ 加 | 系统送的,白用 |
| 顶部 1pt 高光线(模拟玻璃反光) | ❌ 一期不做 | 系统材质已经有玻璃感,加这条边际效益低 |
| 三层阴影 + 内描边(`0 24 60 ... + inset 0 0 0 0.5 ...`) | ❌ 一期不做 | 系统 `NSPopover` / `NSPanel` 自带阴影,够用 |
| 精确字号表(22pt -0.5 tracking 等) | ⚠️ 部分 | KPI 大数字会用,其余用 SwiftUI 默认(`.body` / `.caption`) |
| 自定义动画曲线(`cubic-bezier(0.2, 0.9, 0.3, 1.15)`) | ❌ 一期不做 | 只有刷新旋转保留;其余用 SwiftUI 默认 `.easeOut` |
| 8 种菜单栏形态 + 8 种 HUD 形态 | ❌ 一期不做 | 一期各做 1 种默认,备选形态二期 |
| App 图标正式版(渐变 .icns) | ❌ 占位 | 用 SF Symbol 临时,二期补 |

### 一句话总结

**默认 Liquid Glass 白用 + 两个产品色 + 双语 + 状态色 + 等宽数字 + 必要自绘图表 + SF Symbols。其余靠系统默认。**

## 1. 整体气质

- **关键词**:克制、原生、半透明磨砂、连续曲率(squircle)、信息密度中等。
- **禁用**:Material Design / Bootstrap / Fluent 风格的自造按钮、彩色阴影、霓虹高光、emoji 状态指示。
- **遵循**:macOS 26 Tahoe HIG。所有交互控件(Switch / Picker / DatePicker / SegmentedControl)用系统原生外观,不重绘。

---

## 2. 材质

**全部用系统给的**,不自己叠玻璃效果。

| 场景 | material |
|---|---|
| 菜单栏弹出面板 | `.popover` |
| 桌面悬浮窗 HUD | `.hudWindow` |
| 主窗口侧边栏 | `.sidebar` |
| 卡片 / 分组 | SwiftUI `.regularMaterial` |

SwiftUI 侧:`.background(.regularMaterial)` / `.thinMaterial`。
AppKit 侧:`NSVisualEffectView`,`blendingMode = .behindWindow`。

**一期不做**:顶部 1pt 高光线、内描边。

---

## 3. 字体

- 全部 `-apple-system` / `.systemFont`,不指定具体字族(系统自动切 SF Pro Display / Text)。
- **所有数字** tabular figures:SwiftUI `.monospacedDigit()`,AppKit `NSFont.monospacedDigitSystemFont(...)`。
  - 原因:刷新百分比时不跳动,代价为 0。

### 字号策略

**默认全部用 SwiftUI 内置语义字号**,只在两处用具体数值:

| 场景 | 用什么 |
|---|---|
| 窗口标题 / 节标题 | `.headline`(13pt 600) |
| 正文标签 | `.body`(13pt) / `.callout`(12pt) |
| 次要 caption / 副标 | `.caption`(11pt 次要色) |
| 等宽小字(路径 / 时间戳) | `.system(.caption2, design: .monospaced)` |
| **KPI 大数字**(统计窗口) | `.system(size: 28, weight: .semibold, design: .default).monospacedDigit()`,tracking -0.5 |
| **小标签 eyebrow**(大写如 `CODEX`) | `.system(size: 10.5, weight: .semibold)`,大写 + tracking +0.4 |

> 其余地方不锁字号 / 字重 / tracking,跟随系统。

---

## 4. 颜色

### 4.1 基础语义色(全部用系统色)

| 用途 | Token |
|---|---|
| 主文本 | `.labelColor` / SwiftUI `.primary` |
| 次要文本 | `.secondaryLabelColor` / `.secondary` |
| 三级文本(中文副标) | `.tertiaryLabelColor` |
| 分隔线 | `.separatorColor` |
| 浅色填充 | `.quaternarySystemFill` / `.secondarySystemFill` |
| Accent | `systemBlue`(系统自动) |

### 4.2 产品识别色(**唯一两个写死的颜色**)

| 服务 | 浅色 | 深色 |
|---|---|---|
| Codex | `#6C6C70`(石墨灰) | `#98989D` |
| Claude Code | `#D97757`(桃橙) | `#E68A6E` |

**用在哪**:服务的色点 / 进度环 / 进度条填充 / pill 文字 / 堆叠图块。**不要**用在大面积背景。

**渲染顺序**:Codex 永远第一,Claude Code 第二。

> 实现:`Color("CodexAccent")` / `Color("ClaudeAccent")`,在 `Assets.xcassets` 里配浅深两套。

### 4.3 状态色

| 状态 | 颜色 |
|---|---|
| 剩余 ≥ 20% | 服务自身色 |
| 剩余 < 20% | `systemOrange` |
| 剩余 = 0 / 超限 | `systemRed` |
| live(刷新正常) | `systemGreen` 小点 |
| stale(数据过期) | `systemOrange` 小点 |
| offline(连续失败) | `systemRed` 小点 |

## 5. 双语规则

### 5.1 通用版式

- **英文为主、中文次要**。中文用次要色,字号小一号,跟在 `·` 后面或下一行。
- 不直译,中文要短而地道。已有惯用词:`用量` / `已连接` / `额度` / `查看统计` / `每日用量` / `周额度` / `菜单栏` / `悬浮窗`。

### 5.2 典型组合

| 场景 | 形态 |
|---|---|
| 节标题 | `Menu Bar · 菜单栏` |
| 行内标签 + 副标 | `Codex` 主行 + 下一行 `OpenAI · Plus` 次要色 |
| 描述段落 | 两行,英文一行、中文一行 |
| 按钮 | `Open Statistics · 查看统计` |

### 5.3 实现

- `Localizable.strings` 配 `en` + `zh-Hans` 两套。
- 双语并存的"主标 + 副标"组合,在视图里用 `Text` 拼接:
  ```swift
  Text("Menu Bar")
  + Text(" · ").foregroundStyle(.secondary)
  + Text("菜单栏").foregroundStyle(.secondary)
  ```
- `Settings` 里给"语言"三档:跟随系统 / 中文 / English。

---

## 6. 圆角

**绝大多数用系统默认**(`NSPopover` / `NSPanel` / `Window` 自己就有正确的连续曲率圆角)。只在自绘卡片 / pill 上需要指定:

| 元素 | 半径 |
|---|---|
| 卡片 / 分组 | 10–12pt |
| 行内 pill | 6–8pt |
| 服务 tile(22pt 方块) | 6pt |

实现统一用 `RoundedRectangle(cornerRadius: ..., style: .continuous)`。

---

## 7. 阴影

**全部靠系统**。`NSPopover` / `NSPanel(.floating)` / `Window` 自带阴影,不再叠 `.shadow()`。

唯一例外:**HUD 悬浮窗**用 `NSPanel`,如果默认阴影不够明显,加一层 `.shadow(color: .black.opacity(0.28), radius: 14, y: 10)`。

---

## 8. 动画

**默认用 SwiftUI 系统动画**(`.easeOut` / `.spring()`),不锁曲线。

唯一指定的一处:

| 场景 | 时长 + 曲线 |
|---|---|
| 刷新图标旋转 | 700ms ease-in-out,转 1 圈 |

弹出 / 关闭面板、窗口出现、hover、切换开关,**全部用 SwiftUI / AppKit 系统默认**,不手动调参。

---

## 9. 图标

- **一律 SF Symbols**,不自造 SVG。
- 常用清单:`gauge.medium` / `arrow.clockwise` / `chart.bar.xaxis` / `gear` / `xmark.circle.fill` / `line.3.horizontal` / `circle.fill`(状态点) / `square.and.arrow.up`。
- 菜单栏图标用**模板模式**(`isTemplate = true`),自动适配深浅菜单栏。
- **App 图标**:一期占位(用 SF Symbol 或简单纯色方块),二期补正式 `.icns`(渐变石墨灰 → 桃橙)。
